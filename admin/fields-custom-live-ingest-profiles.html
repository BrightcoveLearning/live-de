Felder für benutzerdefinierte Live-Aufnahmeprofile

Dieses Thema enthält eine Referenz für die Felder, die in Aufnahmeprofilen für Live verwendet werden.

Profilfelder

EIGENTUM Typ Beschreibung Standard
account_id ganze Zahl Die Video-Cloud Konto-ID keiner
name Schnur Der Profilname - muss innerhalb des Kontos eindeutig sein keiner
display_name Schnur Der Anzeigename für das Profil, der auf der Studio-Admin-Seite angezeigt wird - kann der gleiche sein wie name, sollte aber beschreibend sein, um Benutzern zu helfen, die beabsichtigte Verwendung für das Profil zu verstehen keiner
description Schnur Die Profilbeschreibung keiner
model_version ganze Zahl Stellen Sie dies auf ein 1 damit das Profil im Live-Modul angezeigt wird keiner
renditions Reihe Array von Wiedergabeobjekten []
packages Reihe Array von DRM-Paketen - Derzeit wird DRM nicht unterstützt, daher sollte dies immer ein leeres Array sein []

Wiedergabefelder

Die folgenden Tabellen zeigen Felder, die im Anforderungshauptteil zum Aktualisieren von Live-Aufnahmeprofilen verwendet werden können.

EIGENTUM Typ Beschreibung Standard
audio_bitrate ganze Zahl Audiobitrate in Kbit/s (min: 16, max: 1024) match source
audio_codec Schnur Audio-Codec - aac | mp3 Bestimmt durch das Format oder Profil <- in den meisten Fällen aac wird am besten sein / td>
decoder_bitrate_cap ganze Zahl in kbps die maximale Bitrate, die dem Decoder über einen Puffer zugeführt wird keiner
decoder_buffer_size ganze Zahl in kbps die Größe des Puffers, der dem Decoder bei Verwendung von a zugeführt wird bitrate_cap keiner
forced_keyframe_rate Nummer Erzwingt alle X Frames einen Keyframe, lässt jedoch zusätzliche Keyframes für den Szenenschnitt usw. zu. keiner
format Schnur Videoformat - ts, mp4, flv keiner
h264_profile Schnur H.264 hat drei häufig verwendete Profile: Baseline(niedrigste) und Main High.[1-5] baseline
hls_optimized_ts boolesch Für HLS optimierte Zeitsegmente - gilt nur für die HLS-Ausgabe keiner
keyframe_interval ganze Zahl Maximale Anzahl von Frames zwischen Keyframes 250
keyframe_rate schweben Maximale Keyframes pro Sekunde keyframe_interval von 250
label Schnur Bildtyp für Bildwiedergaben - poster oder thumbnail keiner
live_stream boolesch ob dies ein Live-Videostream sein wird keiner
live_dvr_sliding_window_duration ganze Zahl Dauer des Streams, um in Sekunden für die LiveDVR-Bereitstellung verfügbar zu bleiben 300

video_bitrate ganze Zahl in kbps 123 = 123 kbps (min: 64, max: 10000) keiner
video_codec Schnur zB h264, ts Bestimmt durch das Format oder Profil
width ganze Zahl Rahmenbreite in Pixel keiner
