Clip

Mit dem Clip-Endpunkt können Sie das Clip-Ereignis eines vorhandenen Live-Jobs planen. Zum angegebenen Zeitpunkt führt unser System die erforderlichen API-Aufrufe durch, um einen Clip-Job in Ihrem Namen zu starten.

Bedarf

  • Der Live-API-Schlüssel für Ihr Konto.
  • In Ihrem Live-Konto muss die Scheduler-Überschreibung aktiviert sein.

Erstellen Sie einen Clip-Workflow

Beim Erstellen eines Clips müssen Sie sowohl die Clip-Zeitgrenze (im Ausgang Feld) und eine geplante Zeit (time_utc Feld), um die Clip-Anfrage zu stellen.

Die Ausgang -Feld im Anforderungstext unterstützt alle die gleichen Felder des Ausgänge Feld im Standard-VOD-Endpunkt, jedoch unterstützen geplante Clips nur eine einzelne Ausgabe und keine Liste. Sehen VOD-Clipping-Support-Dokumente Einzelheiten zu verfügbaren Feldern.

Normalerweise würden Sie einstellen time_utc um die Endbegrenzung Ihres Clips anzupassen. Die geplante Zeit MUSS nach der Endgrenze des Clips liegen, da sonst die Clipanforderung zum geplanten Zeitpunkt fehlschlägt.

Zeitstempel für die time_utc Feld muss in UTC formatiert als Unix-Zeitstempel in Sekunden sein. Sie können verwenden https://www.unixtimestamp.com/ zur Umwandlung.

Nehmen wir an, wir möchten einen 15-Minuten-Clip planen, der bei 2021-02-09T 13:55:00 + 00:00 beginnt. Wir können die nutzen output.start_time und output.end_time um unsere Clipgrenze als Unix-Zeit in Sekunden anzugeben. (Wieder sehen VOD-Clipping-Support-Dokumente Einzelheiten zu verfügbaren Feldern).

start_time: 2021-02-09T 13:55:00 + 00:00 = 1612878900 end_time: 2021-02-09T 14:10:00 + 00:00 = 1612879800

Da der Scheduler auch eine Zeit benötigt, um die Clip-Anfrage zu planen, können wir den gleichen Wert verwenden wie endzeit da es auch Unix-Zeit in Sekunden ist.

Wir wollen auch ein Videocloud-Video für den Clip erstellen, also binden wir das in die Ausgang. Zum Zeitpunkt der Planung des Workflows wird ein Videocloud-Video erstellt, dessen ID in der Antwort zurückgegeben wird, damit Sie die Video-ID kennen, bevor der Clip erstellt wird. Wenn Sie die Clipanforderung später abbrechen oder sie zum geplanten Zeitpunkt fehlschlägt, wird das Videocloud-Video NICHT aus Ihrem Konto entfernt.

Stellen Sie die folgende Anforderung, um den Workflow zu erstellen

POST https://api.bcovlive.io/v1/scheduler/clip Header: Content-Type: application/json X-API-KEY: Ihr API Key Body: { "job_id": "your Job ID", "description": "My Scheduled Clip", "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800}}

Sie sollten eine ähnliche Antwort erhalten wie

 { "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" }} }, "time_utc": 1612879800 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

Beachten Sie, dass clip.output.videocloud.video.id enthält die ID des neu erstellten Videocloud-Videos. Bei time_utc die Videoquelle wird mit dem aufgenommenen Clip aktualisiert.

Aktualisieren Sie einen Clip-Workflow

Sie können beide aktualisieren Ausgang und time_utc Ihres Clips. Diese können nur aktualisiert werden, wenn "zustand": "ausstehend".

Nach dem vorherigen Beispiel soll der Clip 5 Minuten kürzer als geplant sein. Beim Aktualisieren der Ausgang Feld müssen Sie das vollständige Objekt angeben, nicht nur die vorgenommenen Änderungen. Dies schließt die Videocloud-Video-ID ein, die vom obigen Erstellungsaufruf zurückgegeben wurde.

Beachten Sie, dass wir in diesem Beispiel beide aktualisiert haben endzeit und time_utc um die 5-Minuten-Verkürzung widerzuspiegeln.

PUT https://api.bcovlive.io/v1/scheduler/clip/workflow_id Header: Content-Type: application/json X-API-KEY: Ihr API Key Body: { "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879500}}

Sie erhalten den aktualisierten Workflow in der Antwort.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" }} }, "time_utc": 1612879500 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

Brechen Sie einen Clip-Workflow ab

Sie können den Clip mit a cancel abbrechen LÖSCHEN Anfrage.

Hinweis: Wenn Sie einen Clip-Workflow abbrechen, werden zum Zeitpunkt der Planung erstellte Videocloud-Videos NICHT vom Backend entfernt. Sie müssen verwaiste Videos manuell aus Ihrem Videocloud-Konto entfernen.

LÖSCHEN https://api.bcovlive.io/v1/scheduler/clip/workflow_id Header: X-API-KEY: Ihr API-Schlüssel

Sie erhalten den aktualisierten Workflow in der Antwort.

 { "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "cancelled", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" }} }, "time_utc": 1612879500 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

Benachrichtigungen

Optional können Sie Benachrichtigungen beim Erstellen des Workflows konfigurieren. Sie müssen eine URL für unseren Service angeben POST zu. Wir werden eine Benachrichtigung senden, wenn der Scheduler den Clip-Aufruf tätigt. Sie können auch so konfigurieren, dass Sie eine Benachrichtigung erhalten n Sekunden vor dem Clip. Siehe die Benachrichtigungshandbuch für mehr Details.

POST https://api.bcovlive.io/v1/scheduler/clip Header: Content-Type: application/json X-API-KEY: Ihr API Key Body: { "job_id": "your Job ID", "description": "My Scheduled Clip", "notification_url": "http://example.com/live/clip/callbacks", "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800, "Benachrichtigung": 600}}
