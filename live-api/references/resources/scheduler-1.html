JobStartStop

Mit dem JobStartStop-Endpunkt können Sie die Aktivierung und Deaktivierung eines vorhandenen Live-Jobs planen. Zum angegebenen Zeitpunkt führt unser System die erforderlichen API-Aufrufe durch, um einen Job in Ihrem Namen zu aktivieren und zu deaktivieren.

Bedarf

  • Der Live-API-Schlüssel für Ihr Konto.
  • In Ihrem Live-Konto muss die Scheduler-Überschreibung aktiviert sein.
  • Ein bestehender Live Job mit "statisch": wahr.

Erstellen Sie einen JobStartStop-Workflow

Nehmen wir an, wir möchten ein 2-stündiges Ereignis planen, das bei 2021-02-02T 14:00:00 + 00:00 beginnt.

Zeitstempel müssen in UTC als Unix-Zeitstempel in Sekunden formatiert sein. Sie können verwenden https://www.unixtimestamp.com/ zur Umwandlung.

Beachten Sie, dass die beschriebenen Aktivierungszeitbeschränkungen Hier gelten weiterhin für geplante Aktivierungen. Wir empfehlen daher, eine Zeit einzuplanen, die mindestens 60 Sekunden vor der gewünschten Startzeit liegt.

In diesem Beispiel wählen wir 5 Minuten vor der beabsichtigten Startzeit.

beginne: 2021-02-02T 13:55:00 + 00:00 = 1612274100 Haltestelle: 2021-02-02T 16:00:00 + 00:00 = 1612281600

Stellen Sie die folgende Anforderung, um den Workflow zu erstellen

POST https://api.bcovlive.io/v1/scheduler/jobstartstop Header: Inhaltstyp: application/json X-API-KEY: Ihr API-Schlüsselkörper: { "job_id": "your Job ID", "activation": { "time_utc": 1612274100 }, „Deaktivierung“: { "time_utc": 1612281600 }, „Beschreibung“: „Meine Dienstagsendung“}

Sie sollten eine ähnliche Antwort erhalten wie

{ "account_id": "your Account ID", "activation": { "state": "pending", "time_utc": 1612274100 }, „Deaktivierung“: { "state": "pending", "time_utc": 1612281600 }, „description“: „Meine Dienstagsendung“, „job_id“: „deine Job-ID“, „type“: „jobstartstop“, „workflow_id“: „d44cf29f4f184757ac0995beb8a0097e“}

Aktualisieren Sie einen JobStartStop-Workflow

Sie können die Aktivierungs- und Deaktivierungszeiten eines JobStartStop-Workflows aktualisieren. Diese können nur aktualisiert werden, wenn "zustand": "ausstehend".

Nach dem vorherigen Beispiel möchten wir das Ereignis 30 Minuten früher als geplant stoppen.

PUT https://api.bcovlive.io/v1/scheduler/jobstartstop/workflow_id Header: Inhaltstyp: anwendung/json X-API-KEY: Ihr API-Schlüsselkörper: { "deactivation": { "time_utc": 1612279800 }}

Sie erhalten den aktualisierten Workflow in der Antwort.

{ "account_id": "your Account ID", "activation": { "state": "pending", "time_utc": 1612274100 }, „Deaktivierung“: { "state": "pending", "time_utc": 1612279800 }, „description“: „Meine Dienstagsendung“, „job_id“: „deine Job-ID“, „type“: „jobstartstop“, „workflow_id“: „d44cf29f4f184757ac0995beb8a0097e“}

Brechen Sie einen JobStartStop-Workflow ab

Sie können die Aktivierung und Deaktivierung mit a abbrechen LÖSCHEN Anfrage. Sie können dies nach der Aktivierung tun, um die Deaktivierung einfach abzubrechen.

LÖSCHEN https://api.bcovlive.io/v1/scheduler/jobstartstop/workflow_id Header: X-API-KEY: Ihr API-Schlüssel

Sie erhalten den aktualisierten Workflow in der Antwort.

{ "account_id": "your Account ID", "activation": { "state": "cancelled", "time_utc": 1612274100 }, „Deaktivierung“: { "state": "cancelled", "time_utc": 1612279800 }, „description“: „Meine Dienstagsendung“, „job_id“: „deine Job-ID“, „type“: „jobstartstop“, „workflow_id“: „d44cf29f4f184757ac0995beb8a0097e“}

Benachrichtigungen

Optional können Sie Benachrichtigungen beim Erstellen des Workflows konfigurieren. Sie müssen eine URL für unseren Service angeben POST zu. Wir senden eine Benachrichtigung, wenn der Scheduler die Aktivierungs- und Deaktivierungsanrufe tätigt. Sie können auch so konfigurieren, dass Sie eine Benachrichtigung erhalten n Sekunden vor der Aktivierung und/oder Deaktivierung. Siehe die Benachrichtigungshandbuch für mehr Details.

POST https://api.bcovlive.io/v1/scheduler/jobstartstop Header: Inhaltstyp: application/json X-API-KEY: Ihr API-Schlüsselkörper: { "job_id": "your Job ID", "activation": { "time_utc": 1612274100, "notification": 600 }, „Deaktivierung“: { "time_utc": 1612281600, "notification": 600 }, „Beschreibung“: „Meine Dienstagsendung“, „notification_url“: "http://example.com/live/jobstartstop/callbacks"}
