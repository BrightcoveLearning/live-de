Überblick: Brightcove Live-API

In dieser Übersicht erfahren Sie, wofür die Live-API bestimmt ist und wie Sie sie verwenden können. Zu den in diesem Dokument enthaltenen Themen gehören unterstützte AWS-Regionen und -CDNs, Live-Kanäle und -Ereignisse sowie das Einfügen von zeitgesteuerten ID3-Metadaten in einen Live-Stream.

Einführung

Die Live-API ist eine REST-basierte API, mit der Sie Live-Streaming-Ereignisse erstellen und verwalten können. Zu den optionalen Funktionen gehören:

  • Serverseitige Anzeigeneinfügung ( SSAI)
  • AES-128 Verschlüsselung
  • Erstellen Sie Video-on-Demand-Assets aus Clips aus dem Live-Stream
  • DVR Fähigkeit
  • Mehrere CDNs

Siehe auch die API-Referenz.

Basis-URL

Die Basis-URL für die Live-API ist:

      https://api.bcovlive.io

Überschriften

Alle Anforderungen werden mithilfe eines API-Schlüssels authentifiziert, der Ihnen bei der Einrichtung Ihres Kontos zur Verfügung gestellt wird. Der Schlüssel wird in einem übergeben X-API-KEY Header. EIN Content-Type Header ist auch erforderlich:

      X-API-KEY : YOUR_APIKey
      Content-Type: application/json

Unterstützte CDNs

Die folgenden CDNs werden für Live-Streaming unterstützt:

  • Akamai
  • Wolkenfront

Andere dateibasierte CDNs sollten funktionieren, wurden jedoch nicht getestet und werden nicht aktiv unterstützt.

Kanäle und Veranstaltungszeiten

Es gibt zwei Kaufoptionen für Live:

  • Kaufen Sie Veranstaltungsstunden mit Streaming-Zeit
  • Kaufen Sie Streaming-Kanäle

Sie können auch sowohl Event-Streaming-Stunden als auch Kanäle erwerben. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu den Angeboten zu erhalten.

Wenn Sie sowohl Veranstaltungsstreaming-Stunden als auch Channels erworben haben, müssen Sie Folgendes beachten:
  • Standardmäßig werden Live-Jobs mit den Streaming-Stunden für Ereignisse abgerechnet.
  • Wenn Sie möchten, dass ein Job stattdessen als Channel-Job in Rechnung gestellt wird, müssen Sie Muss Fügen Sie beim Erstellen des Live-Jobs das folgende Feld in den Anforderungshauptteil ein:

    "channel_type": "24x7"

  • Nur-Kanal-Kunden müssen beim Erstellen des Live-Jobs immer das folgende Feld in den Anfragetext aufnehmen:

    "channel_type": "24x7"

  • Kunden, die nur Event-Stunden Streaming-Zeit gekauft haben, müssen nicht einschließen "channel_type": "24x7" in Anfragen.

Token-Authentifizierung

Brightcove bietet die Möglichkeit, Token-Authentifizierung zu Live-Videostream-Wiedergabe-URLs hinzuzufügen. Wenn Sie eine Tokenauthentifizierung hinzufügen möchten, Wenden Sie sich an den Brightcove-Support. Es kann bis zu drei Tage dauern, bis die Tokenauthentifizierung eingerichtet ist.

Die TTL (Time-to-Live) für die Token kann auf einen beliebigen Wert zwischen einer Stunde und 365 Tagen eingestellt werden. Wie lange Sie die TTL einstellen, hängt davon ab, welche Arten von Live-Streams Sie bereitstellen. Beachten Sie jedoch, dass die TTL eine kontoweite Einstellung ist und für alle Live-Streams gilt.

DVR Fähigkeit

Brightcove Live-Streams haben DVR Fähigkeit. Um diese Funktion nutzen zu können, müssen Sie:

  • Verwenden Sie die playback_url_dvr URL für die Wiedergabe
  • Verwenden Sie einen Player, der DVR Fähigkeit

Die DVR-Fähigkeit ist auf 86.400 Sekunden begrenzt.

Die DVR Der Stream bleibt 7 Tage lang verfügbar, nachdem der Live-Stream abgeschlossen ist.

Beachten Sie, dass Brightcove Live eine eigene Konto-ID hat, die von allen anderen Konto-IDs getrennt ist. account_id Bezieht sich in den API-Vorgängen auf Ihre Live-Konto-ID.

Erstellen und Verwalten von Arbeitsplätzen

Mit diesen Vorgängen können Sie einen Live-Job erstellen, die Details abrufen und ihn stoppen. Es gibt auch einen Endpunkt, um einen sofortigen Cue-Point für eine Werbeunterbrechung zu erstellen.

Erstellen Sie einen Live-Job

      POST https://api.bcovlive.io/v1/jobs

Dieser Endpunkt wird verwendet, um Live-Streams über a zu erstellen POST Anfrage. Zusätzlich zur Angabe der Eigenschaften des Live-Streams selbst kann die Anforderung auch VOD-Clips angeben, die aus dem Live-Stream generiert werden sollen (dies kann auch später über das erfolgen Endpunkt). Details zu den Feldern, die in den Anforderungshauptteil aufgenommen werden können, finden Sie in der API-Referenz.

Eingabeprotokoll

Brightcove Live unterstützt mehrere Eingabeprotokolle. Verwenden Sie die protocol Feld im Anforderungshauptteil, wenn Sie den Job erstellen, um den zu verwendenden anzugeben. Unterstützte Werte sind:

  • rtmp(der Standardwert)
  • rtp
  • rtp-fec
  • srt

Das RTMP-Protokoll dient zur Übermittlung eines Streams im FLV-Format. Die anderen Protokolle dienen zur Bereitstellung von MPEG2-TS.

Wenn du benutzt rtp , rtp-fec oder srt , müssen Sie auch angeben a cidr_whitelist (sehen Classless Inter-Domain Routing).

Wenn du benutzt rtmp können Sie eine angeben ip_whitelist für die Eingabe stattdessen, aber dies ist nicht erforderlich.

Beispiel für einen Anforderungshauptteil für einen RTP + FEC-Job:

{
    "live_stream":true,
    "region":"us-west-2",
    "reconnect_time":300,
    "outputs":[
      {
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 800,
        "segment_seconds": 6,
        "keyframe_interval": 90
      }
    ],
    "protocol": "rtp-fec",
    "cidr_whitelist": ["127.0.0.1/32"]
}

Die Live-API Schnellstart führt Sie durch das Erstellen eines Livestream-Auftrags und das Einrichten eines Brightcove-Players zum Abspielen.

Live-Jobs auflisten

      GET https://api.bcovlive.io/v1/jobs

Dieser Endpunkt wird verwendet, um Ihre Live-Streams über a aufzulisten GET Anfrage. Der Endpunkt unterstützt die Paginierung, Sortierung und Suchfilterung. Details zu den Feldern, die in den Anforderungshauptteil aufgenommen werden können, finden Sie in der API-Referenz und einige zusätzliche Informationen finden Sie in Abrufen einer Liste von Live- oder VOD-Jobs.

Holen Sie sich Live-Job-Details

      GET https://api.bcovlive.io/v1/jobs/:jobId

Über diesen Endpunkt können Sie detaillierte Informationen zu einem Live-Stream abrufen, die auch beim ursprünglichen Erstellen des Jobs zurückgegeben werden. Siehe die API-Referenz Einzelheiten zu den Antwortfeldern.

Manuelles Einfügen von Ad Cue Points

      POST https://api.bcovlive.io/v1/jobs/:jobId/cuepoint

Normalerweise sendet Ihr Encoder Cue-Points für Werbeunterbrechungen. Sie können jedoch auch eine sofortige Werbeunterbrechung erstellen, indem Sie eine Anforderung an diesen Endpunkt senden. Siehe die API-Referenz für Details.

Beachten Sie, dass a timecode in der Form DD:HH:MM:SS wird für den Cue-Point benötigt.

Stoppen Sie einen Live-Job

      PUT https://api.bcovlive.io/v1/jobs/:jobId/cancel

Verwenden Sie diesen Endpunkt, um einen Live-Stream sofort zu stoppen. Nach dem Abbrechen kann ein Live-Stream nicht mehr neu gestartet werden. Siehe die API-Referenz für Details.

Clips erstellen

Sie können Video-on-Demand-Clips aus einem Live-Stream erstellen und in einem Video Cloud-Konto speichern oder an den S3-Bucket oder die FTP-Adresse senden. Sie können die Clips definieren, wenn Sie den Live-Stream erstellen, oder sie später mithilfe des unten beschriebenen Endpunkts erstellen. Siehe auch die Clips erstellen leiten.

VOD-Clip erstellen

      POST https://api.bcovlive.io/v1/vods

Die Start- und Endpunkte für die Clips können als Offsets vom Beginn des Streams oder als UNIX-Zeitstempel definiert werden. Details zu den Feldern des Anforderungshauptteils finden Sie in der API-Referenz.

Beachten Sie, dass Clips aus Livestreams erstellt werden können für bis zu 7 Tage nach Beendigung der Veranstaltung.

Holen Sie sich eine Liste der VOD-Jobs (Clip-Jobs)

Eine Liste Ihrer VOD-Jobs für Clips finden Sie unter Abrufen einer Liste von Live- oder VOD-Jobs und der API-Referenz.

Verwaltung SSAI

Mithilfe der serverseitigen Anzeigeneinfügung ( SSAI) können Sie beliebig viele Werbeunterbrechungen in Ihren Live-Stream einfügen. Sie können auch Slate-Assets (VOD-Clips) aufnehmen, um nicht genutzte Anzeigenzeiten mit einer Be-Right-Back-Nachricht oder einer beliebigen Nachricht zu füllen.

Derzeit ein SRT + SSAI Konfiguration wird von Brightcove Live nicht unterstützt. Es ist jedoch möglich, SSAI in einen SRT-Stream mit einzufügen SCTE-35 vom Encoder gesendete Marker. Um zu erfahren, wie Sie SCTE-35-Marker hinzufügen, lesen Sie bitte Ihre Encoder-Dokumentation.

Weitere Details zur Einrichtung SSAI kann gefunden werden in Serverseitige Anzeigeneinfügung mit Brightcove Live-API und der API-Referenz.

Konfigurationen für Kontoanzeigen abrufen

      GET https://api.bcovlive.io/v1/ssai/applications/:account_id

Mit diesem Endpunkt können Sie alle Anzeigenkonfigurationen abrufen, die für ein Konto eingerichtet wurden. Details zu den Antwortfeldern finden Sie in der API-Referenz.

Anzeigenkonfiguration erstellen

      POST https://api.bcovlive.io/v1/ssai/application

Erstellen Sie eine Anzeigenkonfiguration, die definiert, wie Anzeigen abgerufen werden für SSAI. Details zu den Feldern des Anforderungshauptteils finden Sie in der API-Referenz.

Anzeigenkonfiguration abrufen

      GET https://api.bcovlive.io/v1/ssai/application/:application_id

Verwenden Sie diesen Endpunkt, um die Details einer von Ihnen erstellten Anzeigenkonfiguration abzurufen. Details zu den Antwortfeldern finden Sie in der API-Referenz.

Anzeigenkonfiguration aktualisieren

      PUT https://api.bcovlive.io/v1/ssai/application/account/:application_id

Aktualisieren Sie die Details einer Anzeigenkonfiguration. Details zu den Feldern des Anforderungshauptteils finden Sie in der API-Referenz.

Holen Sie sich Slate Media Source Assets

      GET https://api.bcovlive.io/v1/ssai/slates/:ACCOUNT_ID

Rufen Sie die Slate Media-Assets ab, die für ein Konto definiert wurden. Slate Media-Assets werden verwendet, um die Werbeunterbrechungszeit zu füllen, die nicht von Anzeigen ausgefüllt wird. Details zu den Antwortfeldern finden Sie in der API-Referenz.

Ingest Slate Media Source Asset

      POST https://api.bcovlive.io/v1/ssai/slates

Fügen Sie ein Medienelement für Slates hinzu, um die nicht ausgefüllte Werbeunterbrechungszeit zu füllen. Details zu den Feldern des Anforderungshauptteils finden Sie in der API-Referenz.

Löschen Sie das Slate Media Source Asset

      DELETE https://api.bcovlive.io/v1/ssai/slates/:SLATE_MSA_ID

Löscht ein Slate-Media-Asset.

Statische Einstiegspunkte

Die SEP-Funktion (Static Entry Points) ermöglicht einen lang laufenden Live-Job, der aktiviert und deaktiviert werden kann, während die Einstiegspunkt-URLs und Wiedergabe-URLs statisch und wiederverwendbar bleiben. Mit dieser Funktion können Kunden ihren Encoder in ihren Einrichtungen oder vor Ort konfigurieren und ihre eigene Planungslogik für Live-Kanäle oder -Programme erstellen. Sehen Statische Einstiegspunkte für Details.

Bildunterschriften

Wenn sich Untertitel innerhalb des h264-Eingangssignals befinden (im Paket user_data korrekt signalisiert), werden diese an die h264-Ausgänge weitergeleitet.

Wenn Sie einen Broadcast-Elementar-Live-Encoder verwenden, können Sie Beschriftungen von SDI (EIA-608/CEA-608) oder anderen Quellen (SCTE-20, SCC, Teletext, DVB-Sub, Hilfs-, ARIB, TTML, SCTE-27, STL, SRT, SMI) abrufen und sie in den h264-Stream abrufen, den Sie uns senden. Andere Encoder mit Broadcast-Qualität können dies wahrscheinlich auch, aber wir haben sie nicht offiziell getestet.

Als aktuelle Einschränkung zeigt Safari die CC-Steuerelemente für Live-Streams an, auch wenn keine Untertitel angezeigt werden.

Fügen Sie zeitgesteuerte ID3-Metadaten ein

Diese Informationen wurden nach verschoben Fügen Sie zeitgesteuerte ID3-Metadaten ein.

Einschränkungen

  • Damit Live-Jobs, die mit der API erstellt wurden, angezeigt werden und nicht im Live-Modul verwendet werden können, müssen Sie das videocloud Objekt beim Erstellen des Jobs in den Anforderungstext aufnehmen.

    Beispielsweise:

    {
  "live_stream": true,
  "region": "eu-central-1",
  "reconnect_time": 1800,
  "live_sliding_window_duration_ms": 0,
  "outputs": [
    { "label": "hls720p", "live_stream": true, "width": 1280, "height": 720, "video_codec": "h264", "h264_profile": "high", "video_bitrate": 2100, "segment_seconds": 4, "keyframe_interval": 60 }
    ,
    { "label": "hls540p", "live_stream": true, "width": 960, "height": 540, "video_codec": "h264", "h264_profile": "main", "video_bitrate": 1500, "segment_seconds": 4, "keyframe_interval": 60 }
    ,
    { "label": "hls360p", "live_stream": true, "width": 640, "height": 360, "video_codec": "h264", "h264_profile": "main", "video_bitrate": 800, "segment_seconds": 4, "keyframe_interval": 60 }
  ],
  "videocloud": {
    "video":
      { "name": "live event UI", "description": "live event UI", "long_description": "", "tags": [], "reference_id": "", "state": "ACTIVE" }
    }
  }
  • Die anfängliche Verbindung vom Encoder liefert die Bandbreiteninformationen, die mit der Live-Wiedergabeliste erstellt werden sollen. Wenn die anfängliche Verbindung niedrig ist, behält die Wiedergabeliste die gleichen Informationen in der Wiedergabeliste bei, selbst wenn die Jobkonfiguration eine hohe Ausgabe hatte, bis Folgendes ausgeführt wird:
    • Encoder wird neu gestartet
    • Möglicherweise muss auch der CDN-Cache geleert werden
  • Derzeit ist die Framerate für Eingabestreams auf 30 FPS begrenzt. Wenn Sie an einer höheren Bildrate interessiert sind, wenden Sie sich bitte an den Support.
  • Standardmäßig ist die Auflösung des Eingabe-Streams auf 1080p begrenzt.
  • Beim Trennen und erneuten Verbinden müssen die Stream-Einstellungen gleich bleiben. Änderungen an der Anzahl der Audiokanäle, Auflösungen oder Codec-Einstellungen führen zu unvorhersehbarem Verhalten.
  • Obwohl Sie DASH und MP4 für Remote-Quellen für Video Cloud-Videos hinzufügen können, unterstützt Live derzeit HLS nur.
  • Für Eingabestreams wird nur AAC-Audio unterstützt.
  • Maximal 5 aktiv Warten, nicht gestartet Jobs sind jederzeit erlaubt.

    Zusätzliche Einschränkungen für gleichzeitige Jobs:

    • Die Anzahl der channel (24x7) Jobs sind auf 0 oder eine niedrige Anzahl pro Region begrenzt (abhängig vom Kontotyp).
    • Die Anzahl der gleichzeitig Laufen event Arbeitsplätze sind regional begrenzt, in der Regel auf 100.
    • Die Anzahl der gleichzeitig Warten auf eine Verbindung event Jobs ist auf 5 begrenzt.
    • Die Anzahl der SEP-Jobs pro Region ist auf 3 oder 10 begrenzt (siehe Unterstützte AWS-Regionen).

    Jedes dieser Limits kann vom Support auf Kontoebene angepasst werden. Wenden Sie sich an Ihren Account Manager, wenn Sie zusätzliche Kapazität benötigen.

  • Die "RTMP" -Adresse wird als zurückgegeben stream_url Bei Live-Jobs handelt es sich um einen Akamai HD Live-Stream, nicht um einen älteren FMS-RTMP-Stream. Er wird von älteren Versionen von Internet Explorer nicht unterstützt.
  • Live-Streams werden über HTTPS bereitgestellt. Wenn Ihr Brightcove-Konto nicht für HTTPS aktiviert ist, kann der Brightcove-Player den Live-Stream nicht laden. Wenn in Ihrem Konto die HTTPS-Unterstützung für die Ursprungsbereitstellung nicht aktiviert ist, wenden Sie sich bitte an Wenden Sie sich an den Brightcove-Support Aktivieren der HTTPS-Unterstützung für Origin Serving, um Wiedergabeprobleme zu vermeiden.
  • Bei Verwendung einer Transmux-Wiedergabe innerhalb eines HLS-Ausgangs mit mehreren Bitraten segment_size kann beim Transmuxen einbezogen werden, sollte aber so eingestellt werden, dass es ein Vielfaches der ist GOP Größe des Eingabestreams. Wenn die Eingabe also 30 fps mit Keyframes alle 60 Frames beträgt, wird die GOP Die Größe beträgt 2 Sekunden und die Segmentgröße sollte ein Vielfaches von 2 sein. Wenn Sie dies nicht tun, haben die Stream-Segmente unterschiedliche Größen.

    Ebenfalls, keyframe_interval sollte nicht an allen Ausgängen angegeben werden.

  • Wenn Sie Ihren eigenen FTP- oder S3-Ursprungsort verwenden, muss Ihr CDN so konfiguriert sein, dass es auf Ihren Ursprungsort zurückgreift. Das Brightcove Live-System überprüft nicht, ob die Ursprungsorte für die in der Auftragsanforderung angegebenen CDNs vorhanden sind.
