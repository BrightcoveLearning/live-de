Schnellstart: Erstellen Sie einen Live-Stream mit der Brightcove Live-API

Dieses Tutorial führt Sie durch die Erstellung eines Live-Stream-Jobs mithilfe der Live-API.

Was du brauchst

Um dieses Tutorial abzuschließen, benötigen Sie Folgendes.

  • Ein Konto, das für die Live-API eingerichtet wurde
  • Ein API-Schlüssel für die Live-API (wird Ihnen bei der Einrichtung des Kontos zur Verfügung gestellt)
  • Ein Video Cloud-Konto (nicht erforderlich, um den Live-Stream-Job zu erstellen, aber erforderlich, um die Wiedergabe im Brightcove Player einzurichten)
  • Wenn Sie die verwenden CMS-API Um das Video Cloud-Video anstelle von Studio zu erstellen, benötigen Sie Client-Anmeldeinformationen mit Video-/Lese- und Video-/Schreibberechtigungen. Wenn Sie mit dem Abrufen von Client-Anmeldeinformationen nicht vertraut sind, lesen Sie den Authentifizierung Abschnitt unten.
  • Die Möglichkeit, REST-API-Anforderungen entweder über zu stellen cURL oder ein REST-Client wie Schlaflosigkeit oder Postbote.

    Um Ihnen zu helfen, haben wir einige Importe von Schlaflosigkeit und Postboten vorbereitet, die Sie für die Übung verwenden können (bei Brightcove Learning Services bevorzugen und verwenden wir Schlaflosigkeit, aber Postman funktioniert auch, wenn Sie dies haben):

    In jeder der Zip-Dateien finden Sie eine Readme-Seite, die Ihnen weiterhilft.

    • Import von Schlaflosigkeitsarbeitsbereichen
    • Briefträgersammlung und Umgebungsimport
  • Um einen tatsächlichen Live-Stream zu erstellen, benötigen Sie entweder einen Software- oder einen Hardware-Encoder für das Live-Streaming. Die folgenden Anweisungen enthalten Schritte zum Einrichten des Livestreams mit Wirecast ( Melden Sie sich für eine Testversion an). Wenn Sie einen anderen Encoder verwenden, ist der Prozess anders, sollte jedoch in Bezug auf die Informationen, die Sie aus der Live-API-Antwort benötigen, ähnlich sein.

Anmeldeinformationen für die CMS-API abrufen

Um die zu verwenden CMS-API Sie benötigen die richtigen Anmeldeinformationen.

Der einfachste Weg, um Anmeldeinformationen zu erhalten, ist in den meisten Fällen der Abschnitt Studio Admin API-Authentifizierung (erfordert Administratorrechte für Ihr Konto). Sehen Verwalten von API-Authentifizierungsanmeldeinformationen für Details. In diesem Fall sind die Berechtigungen, die Sie benötigen, für Beziehungen teilen - Sie benötigen sowohl Lese- als auch Schreibberechtigungen:

Wenn die von Ihnen benötigten Berechtigungen in Studio nicht verfügbar sind oder Sie sie lieber direkt von der OAuth-API beziehen möchten, verwenden Sie Ihre Wahl der Kundenanmeldeinformationen abrufen unten aufgeführten Dokumente. Unabhängig davon, welche Option Sie auswählen, müssen Sie nach den richtigen Betriebsberechtigungen fragen. Folgendes kann mit cURL oder Postman verwendet werden, um die richtigen Berechtigungen zu erhalten:

    "operations": [
    "video-cloud/videos/read",
    "video-cloud/videos/create",
    "video-cloud/videos/update"]
  • OAuth: Abrufen von Client-Anmeldeinformationen mithilfe von cURL
  • OAuth: Abrufen von Client-Anmeldeinformationen mithilfe von Postman

Richten Sie das Video Cloud-Video ein

Nachdem wir die Live-Stream-Informationen haben, richten wir ein Remote-Video in Video Cloud ein, damit wir es in einem Brightcove-Player abspielen können.

In diesen Schritten richten wir das Video mit dem ein CMS API Dies kann aber auch in Video Cloud Studio erfolgen, wenn Sie dies bevorzugen - siehe die alternative Schritte unten.

Wenn Sie Insomnia oder Postman verwenden, müssen Sie die OAuth2-Authentifizierung mithilfe Ihrer Client-Anmeldeinformationen einrichten. Lesen Sie eine der folgenden Anleitungen, wenn Sie Hilfe benötigen:

  • Verwenden Sie Insomnia für API-Anfragen
  • Verwenden Sie Postman für API-Anfragen

  1. Zunächst müssen Sie ein neues Video erstellen, indem Sie ein POST Anfrage zu:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

  2. Der Anforderungshauptteil kann mehrere Metadaten enthalten (siehe die API-Referenz für Details), aber das einzige, was benötigt wird, ist a name::

    {
    "name": "some name for your live stream"
}

  3. Die Antwort enthält alle Video-Metadaten, aber das wichtige Feld ist das id, das Sie zum Hinzufügen des Remote-Assets benötigen.

  4. Um den HLS-Stream hinzuzufügen, erstellen Sie einen weiteren POST Anfrage zu:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id/assets/hls_manifest

  5. Diesmal lautet der Anfragetext:

    {
    "remote_url": "playback_url for your live stream"
}

Video zu einem Player hinzufügen

Jetzt erstellen wir eine einfache HTML-Seite und fügen einen Player mit dem gerade erstellten Video hinzu.

Auch hier verwenden wir Studio, um den Player-Veröffentlichungscode zu erhalten - Sie können dies auch mit dem Spielerverwaltungs-API wenn Sie es vorziehen.
  1. Erstellen Sie eine einfache HTML-Seite, um den Live-Stream-Player zu testen. Sie können jeden anderen oder gar keinen Inhalt hinzufügen.
  2. Wählen Sie im Studio Media-Modul das in den vorherigen Schritten erstellte Video aus und klicken Sie auf Veröffentlichen und einbetten> Web Player::
  3. Wählen Sie im Dialogfeld "Veröffentlichen" einen Player aus (der Brightcove-Standardplayer ist in Ordnung):
  4. Scrollen Sie nach unten, stellen Sie die Größe des Players wie gewünscht ein und klicken Sie dann in den Codeblock, um den Standard-Player-Code auszuwählen. Kopieren Sie diesen Code und fügen Sie ihn in den Text Ihrer HTML-Seite ein:
  5. Speichern Sie Ihre HTML-Seite auf einem Remote-oder lokalen Webserver, damit Sie sie durchsuchen können.
  6. Öffnen Sie die Seite in Ihrem Webbrowser. Da wir noch nicht mit dem Dämpfen begonnen haben, wird ein Medienfehler angezeigt:

  4. In dem Remote-Video hinzufügen Fügen Sie im Dialogfeld einen Videonamen hinzu und kopieren Sie den Wert für und playback_url von der Live-API-Antwort in das URL-Feld. Klicken Sie dann auf Formatvariante hinzufügen und auf Speichern:

Verwenden Sie Wirecast, um ein Ereignis zu streamen

In diesem Abschnitt konfigurieren und verwenden wir den Telestream Wirecast-Encoder, um ein Live-Streaming-Ereignis zu unterstützen. Die Telestream Wirecast-Software ist eine Desktop-Anwendung, die Eingaben von Ihrer Kamera erfasst und einen Stream erzeugt, der von einem CDN bereitgestellt werden kann. Beachten Sie, dass Hardware- und andere softwarebasierte Codierungslösungen verfügbar sind, die möglicherweise besser für die Übertragung Ihres Live-Streaming-Ereignisses geeignet sind. Sie können eine Testversion der Wirecast-Software von der herunterladen Telestream-Website.

Hinweis: Sie sollten die Wirecast-Software installiert und eine Kamera an Ihren Computer angeschlossen haben, bevor Sie fortfahren.

Führen Sie die folgenden Schritte aus, um Wirecast für ein Live-Ereignis zu konfigurieren.

  1. Öffnen Sie Wirecast.
  2. Bewegen Sie den Mauszeiger über die +, klicken Geräte erfassen und wählen Sie dann Ihre Kamera aus.
  3. Klicken Sie auf die Pfeilschaltfläche ( ), damit die Kamera die Live-Aufnahme macht.
  4. Drücke den Strom Taste ( ). Sie werden aufgefordert, die Ausgabeeinstellungen zu konfigurieren.
  5. Stellen Sie die Ziel zu RTMP-Server. Klicken OK.
  6. Für die Die Anschrift , benutze die stream_url von der Live-API zurückgegeben (Schritt 3).
  7. Verwenden Sie für den Stream live.
  8. (Optional) Klicken Sie auf, um zusätzliche Ausgabestreams zu erstellen Hinzufügen... Wenn Sie zusätzliche Ausgabestreams erstellen, möchten Sie möglicherweise ein eigenes benutzerdefiniertes Codierungsprofil erstellen, um die Ausgabebitrate zu steuern. Stellen Sie sicher, dass die Strom Name spiegelt die neue Bitrate wider.
  9. Klicken OK um die Ausgabeeinstellungen zu speichern.
  10. Drücke den Strom Taste ( ). Der Live-Stream sollte beginnen.

Fazit

Sie können jetzt eine Live-Streaming-Sitzung erstellen. Sie müssen Ihrer Encoder-Konfiguration Folgendes hinzufügen:

  • stream_url: Ergänzen Sie die stream_url von Ihrem Job (die rtmp Adresse)
  • stream_name: Der Streamname muss sein alive

Sobald Sie den Live-Stream gestartet haben, sollte er in dem Brightcove-Player angezeigt werden, den Sie in den vorherigen Schritten erstellt haben.
