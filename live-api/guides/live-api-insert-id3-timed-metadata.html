Live-API: Fügen Sie zeitgesteuerte ID3-Metadaten ein

In diesem Thema wird erläutert, wie Sie zeitgesteuerte ID3-Metadaten mithilfe der Live-API in Ihren Live-Stream einfügen.

Überblick

Zeitgesteuerte ID3-Metadaten werden verwendet, um Daten mit einem Live-Stream zu einem bestimmten Zeitpunkt zu senden. Um zeitgesteuerte ID3-Metadaten für einen laufenden Job einzufügen, erstellen Sie a POST Anfrage zu:

https://api.bcovlive.io/v1/jobs/JOB_ID/id3tag

Beispielanforderungshauptteil

{
 "id3_tag": {
 "name": "cp1",
 "value": "Some information I want to send",
   "timecode": "15:50:49:16"
 }
}
Hinweis:
  • Nur id3_tag Objekt kann in einer Anfrage eingereicht werden
  • Wenn Sie die verwenden timecode Eigenschaft speichert der Job nur die letzte Anforderung zum Einfügen
  • Wenn Sie die verwenden timecode Eigenschaft muss der Encoder einen SMPTE-formatierten (HH: MM: SS: FF) Timecode senden, der in der Eigenschaft tc über OnFI gespeichert ist
  • Software-Encoder wie Wirecast und OBS unterlassen Sie Unterstützung des Sende-Timecodes über OnFI-Pakete im RTMP-Stream
  • Elementare Hardware-Encoder tun Unterstützung des Sende-Timecodes über OnFI-Pakete im RTMP-Stream
  • ID3v2 gibt Tag an name kann bis zu 4 Zeichen lang sein.
  • value kann bis zu 256 KB groß sein.
  • Timecode ist optional - wenn weggelassen, erfolgt das Einfügen sofort.

Beispielantwort

{
"id": "JOB_ID",
"id3_tag": {
"tag_name": "BCOV",
"tag_value": "my value"
}
}

Gibt es SMPTE-Timecodes in der Eingabe?

Nicht alle Encoder senden SMPTE-Timecodes. Um zu überprüfen, ob sie in Ihrer Eingabe vorhanden sind, können Sie in der VOD-Chunklist nach Tags wie suchen #VOD-TIMING:timecode=15:18:36:01,wallclock=1549047672441,tsvideo=25400,tsaudio=-1. Die Vod-Chunklist-URL sollte unter verfügbar sein output_media_files[0].playback_url_vod wenn Sie die Jobinformationen abrufen.

Analysieren Sie eine Live-Wiedergabeliste, um den Timecode zu erhalten

Wenn Sie einen Brightcove Live-Job erstellen, wird eine Standard-Master-Wiedergabeliste zurückgegeben. Beispielsweise:

https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist.m3u8

Wir benötigen die VOD-Master-Wiedergabeliste, für die wir im letzten Teil unserer URL-Erweiterung als "_vod" hinzufügen müssen playlist_vod.m3u8

https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8

curl -i "https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8"
