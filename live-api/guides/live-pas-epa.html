PAS und EPA befinden sich derzeit in der Version mit eingeschränkter Verfügbarkeit. Wenn Sie daran interessiert sind, diese Dienste zu nutzen, wenden Sie sich an Ihren Kundenbetreuer.

Einführung

Für einen Live-Job kann entweder PAS oder EPA aktiviert werden, um den von DRM bereitgestellten Schutz zu erweitern. PAS bietet eine begrenzte Anzahl von Einschränkungen: Beschränken der Wiedergabe auf einen bestimmten Benutzeragenten oder Begrenzen der Anzahl von IP-Adressänderungen oder der Gesamtverwendung. Im Live-Fall kann ein Kunde auch die DRM-CRT für diesen Benutzer überschreiben. Diese Einschränkungen sind direkt in der Wiedergabeautorisierung festgelegt JWT.

EPA bietet eine viel umfassendere Reihe von Einschränkungen. Ein Satz dieser Einschränkungen wird über die Playback Rights Management-API konfiguriert und als Wiedergaberechte bezeichnet id. Diese id kann dem Video zugeordnet werden (mit dem playback_rights_id Attribut unten gezeigt) oder in der enthalten JWT. Der zweite Satz beinhaltet Einschränkungen auf Benutzerebene wie Parallelitätsbeschränkungen und Anzahl der Geräte. Diese werden direkt in den JWT-Ansprüchen angegeben (die auch eine Benutzer-ID enthalten müssen [vuid Anspruch]).

In beiden Modi muss die Jobanforderung a enthalten videocloud.video Attribut, damit ein Video Cloud-Video erstellt und mit dem Live-Job verknüpft wird. Das playback_rights_id Das Attribut kann im Video Cloud-Video aktualisiert werden, um die Wiedergabebeschränkungen über das zu ändern CMS API.

Sie müssen auch Erstellen Sie eine JWT mit Ansprüchen, die für PAS oder EPA geeignet sind, je nachdem, welche Sie verwenden. Das JWT muss in allen Lizenzserveranforderungen enthalten sein. Wenn Sie Ihren eigenen Player implementieren, müssen Sie diesen Wert entweder in a übergeben BCOV-Auth Header oder bcov_auth Abfrageparameter (zusätzlich zum DRM-Token).

Erstellen Sie einen Live-Job mit EPA / PAS

Die folgenden neuen Felder werden verwendet, um einem Live-Job EPA und / oder PAS hinzuzufügen:

Neue Felder für die drm Objekt:

require_playback_token(boolescher Wert)
Aktiviert PAS. Playback Auth JWT ist erforderlich, wenn Sie eine Lizenz anfordern.
check_playback_rights(boolescher Wert)
Aktiviert EPA. Playback Auth JWT ist erforderlich, wenn Sie eine Lizenz anfordern. Verwendet die mit Video oder JWT verknüpfte Wiedergaberechten-ID, um Einschränkungen durchzusetzen.

Neues Feld für die videocloud.video Objekt:

Beachten Sie, dass dieses Feld mithilfe der CMS-API zum Video hinzugefügt werden muss nach Sie erstellen den Live-Job, da er beim Erstellen des Videos nicht hinzugefügt werden kann.
playback_rights_id(String)
Verknüpft die angegebenen EPA-Wiedergaberechte mit Video.

CMS-API-Anfrage

Senden Sie eine, um dem Video die ID für die Wiedergaberechte hinzuzufügen Video aktualisieren Anforderung an die CMS-API mit dem folgenden Anforderungshauptteil:

{
	"playback_rights_id": "your playback_rights_id"
}

Player-Implementierung

Wenn Sie den Brightcove-Webplayer oder eines der nativen Brightcove-Player-SDKs zur Bereitstellung des Live-Streams verwenden, ist die Implementierung für PAS oder EPA dieselbe wie für VOD. Siehe eine der folgenden Angaben:

PAS

  • Verwenden der Wiedergabeautorisierung DRM mit dem Brightcove Player
  • Verwenden von DRM für die Wiedergabeautorisierung mit den nativen SDKs

EPA

  • Verwenden von Wiedergaberechten mit Brightcove Player
  • Verwenden von Wiedergaberechten mit den nativen SDKs

Verwandte Themen

  • PAS
  • EPA
  • Erstellen von JSON-Web-Tokens (JWT)
  • Verwenden der Wiedergabeautorisierung DRM mit dem Brightcove Player
  • Verwenden von DRM für die Wiedergabeautorisierung mit den nativen SDKs
  • a href = "https://apis.support.brightcove.com/playback-rights/guides/using-epa-with-bcp.html"> Verwenden von Wiedergaberechten mit Brightcove Player
  • Verwenden von Wiedergaberechten mit den nativen SDKs
