Verwalten von Anmeldeinformationen für die Live-API

Dieses Thema bietet einen Überblick über die Verwaltung von Anmeldeinformationen verschiedener Typen für die Live-API.

Einführung

Es gibt verschiedene Arten von Anmeldeinformationen, die in Verbindung mit Ihren Live-API-Jobs verwendet werden können:

  • Anmeldeinformationen für FTP, S3, Video Cloud oder Zencoder, die das Live-System benötigt, um Clips an das von Ihnen gewählte Ziel zu senden
  • RTMP-Anmeldeinformationen, die verhindern, dass jemand, der die RTMP-Adresse Ihres Live-Steam erhält, diese wiedergibt
  • OAuth2 oder grundlegende Berechtigungsnachweise, die das Live-System benötigt, um Benachrichtigungen an Ihren Benachrichtigungshandler zu senden

Mit den Endpunkten für Anmeldeinformationen für die Live-API können diese Anmeldeinformationen erstellt und verwaltet werden, auf die Sie beim Erstellen von Live- oder VOD-Clipjobs anhand ihrer Beschriftungen verweisen können.

Ausführliche Informationen zu den API-Vorgängen für Anmeldeinformationen finden Sie in der API-Referenz.

Anmeldeinformationstypen

In der folgenden Tabelle sind die derzeit unterstützten Anmeldeinformationstypen und ihre Verwendung aufgeführt.

Anmeldeinformationstypen
credential_type Wert Beschreibung / Anwendungsfall
ftp Anmeldeinformationen für die FTP-Site, an die Clips gesendet werden sollen
http Grundlegende Berechtigungsnachweise für eine http-Adresse, an die Benachrichtigungen gesendet werden sollen
oauth Anmeldeinformationen für die OAuth2-Authentifizierung sind für die Adresse erforderlich, an die Benachrichtigungen gesendet werden sollen
rtmp Fügen Sie dem Endpunkt, zu dem Ihr Encoder eine Verbindung herstellt, eine Authentifizierung hinzu, damit jeder, der über die RTMP-URL verfügt, nicht einfach eine Verbindung herstellen kann
s3 Anmeldeinformationen für einen S3-Bucket, an den Clips gesendet werden sollen
videocloud OAuth-Anmeldeinformationen für Ihr Video Cloud-Konto zum Senden von Clips (müssen die für Dynamic Ingest erforderlichen Berechtigungen enthalten)
zencoder Authentifizierung für Ihr Zencoder-Konto, an das Clips gesendet werden sollen

Anmeldeinformationen Operationen

Die folgende Tabelle zeigt die Endpunkte für die Verwaltung von Anmeldeinformationen.

Endpunkte für Anmeldeinformationen
Endpunkt Methode Beschreibung
/v1/credentials GET Listen Sie alle Anmeldeinformationen für das Konto auf
/v1/credentials POST Erstellen Sie einen neuen Berechtigungsnachweis für das Konto
/v1/credentials/{credential_id} PUT Aktualisieren Sie einen Berechtigungsnachweis für das Konto
/v1/credentials/{credential_id} DELETE Löschen Sie einen Berechtigungsnachweis für das Konto

Die folgende Tabelle zeigt die Felder für Anmeldeinformationen.

Anmeldeinformationsfelder
Field Type Required Allowed Values Default Description
credential_default_for_type boolean no true | false false Want this credential to be default for the user for this type of credentials?
credential_label string yes     User identifiable name for the credentials
credential_type enum yes ftp | http | oauth | rtmp | s3 | videocloud | zencoder   The type of credential
credential_public string yes     The username, public key, or client id, depending on the credential type
