Live-API: VOD von Live-Event

In diesem Thema erfahren Sie, wie Sie nach Abschluss des Ereignisses aus einem Live-Streaming-Ereignis ein Video-on-Demand (VOD) erstellen.
Live to VOD kann nicht mit verschlüsselten Streams verwendet werden und gibt beim Erstellen eines Live-Jobereignisses einen Fehler zurück.

Einführung

Bei kürzeren Live-Streaming-Events möchten Sie vielleicht eine Aufzeichnung des Events als Video-on-Demand (VOD) in Ihrem Video-Cloud Konto nach Abschluss der Veranstaltung. Sie können dies einfach tun, indem Sie beim Konfigurieren eine Konfiguration für den VoD hinzufügen Erstellen Sie Ihren Live-Job.

Mit Fast Live to VOD können Clips schnell und segmentgenau veröffentlicht werden, während ein für den Kunden vollständig transparenter Backend-Prozess die Clipgrenzen bereinigt und erneut veröffentlicht.

Das Ergebnis ist eine schnellere Markteinführung eines Clips, der automatisch mit den gewünschten Clipgrenzen neu veröffentlicht wird.

Wichtig::

  • Wenn das Live-Streaming-Ereignis abgeschlossen ist, stoppen Sie Ihren Encoder, aber unterlassen Sie stornieren die Arbeit. Warten Sie, bis der VOD-Clip erstellt wurde, bevor Sie den Auftrag abbrechen. Andernfalls wird er nicht erfolgreich erstellt.
  • Die Erstellung des VOD beginnt erst am reconnect_time denn der Live-Job ist abgelaufen.
  • Wenn die Dauer des Live-Events länger als 24 Stunden ist, werden nur die letzten 24 Stunden im VOD erfasst.

Schnelle VOD-Option

Mit der schnellen VOD-Option können Sie die VOD-Version des Live-Events schneller abrufen. Fast Live to VOD veröffentlicht (unter Verwendung eines Transmux) Live-Wiedergaben segmentgenau.

Die output Felder für schnelles VOD sind:

  • live_to_vod einstellen true
  • mode(erforderlich für schnelles VOD) - für schnelles VOD wird der Wert sein instant
  • playlist_label(optional) - Gilt nur für Sofortmodus-VODs. Gibt an, welche Wiedergabeliste zum Erstellen einer VoD-Ausgabe (vom Live-Job) verwendet werden soll. Nur erforderlich, wenn bei der Erstellung von Live-Jobs benutzerdefinierte Wiedergabelisten definiert wurden.

Implementierungen

Wenn Sie den Live-Job erstellen

Der einfachste Weg, Live in VOD zu implementieren, besteht darin, das einzuschließen "live_to_vod": true Feld im Anforderungshauptteil, wenn Sie den Job erstellen, entweder in:

  • die oberste Ebene videocloud Objekt (um den VOD als Remote-Asset zu erstellen)
  • ein Ausgabeobjekt, das an Video Cloud oder an S3 ausgibt

Senden an Video-Cloud

Beachten Sie, dass es zwei Möglichkeiten gibt videocloud Objekte werden verwendet, wenn Sie einen neuen Live-Job erstellen:

  1. Einbeziehung videocloud in die Felder auf Jobebene wird ein Video im Video Cloud-Konto erstellt, das das Live aufnehmen wird streamen Sie als Remote-Asset und erstellen Sie das VOD in Ihrer Video Cloud-Bibliothek: 
    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
    
      "videocloud": {
        "video": {
          "name": "test_live_stream"
        },
        "live_to_vod": true
        }

  2. Einschließlich der videocloud Objekt in den Eigenschaften von an outputs Clip oder vollständiges VOD-Objekt erstellt a Video-Cloud Video für den Clip - der Clip wird aufgenommen in Video-Cloud und wird Teil der VOD-Bibliothek:

    {
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 2400,
  "live_dvr_sliding_window_duration": 30,
	"videocloud": {
		"video": {
			"name": "test_live_stream"
		}
	},
  "outputs": [
    { "videocloud": {
        "video": {
            "name": "video name",
        },
        "ingest": {
            "profile": "valid-ingest-profile-name"
        }
    },
    "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
    }
  ]
}

    Hinweis: Lassen Sie das Feld Aufnahmeprofil weg, um das Standard-Aufnahmeprofil für das Konto zu verwenden.

Sie können auch die Videowolke Objekt innerhalb eines Ausgabeobjekts (in diesem Fall müssen Sie nicht live_to_vod Gebiet):

{
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 600,
  "ad_insertion": false,
  "outputs": [
    { OTHER_JOB_OUTPUT_OBJECT },
    { OTHER_JOB_OUTPUT_OBJECT_2 },
    { "videocloud": {
        "video": {
            "name": "video name",
            "description": "video description",
            "long_description": "long description",
            "tags": [
            "tag1", "tag2"
            ],
            "schedule":
            { "starts_at": "2017-01-01T12:00:00-04:00", "ends_at": "2099-01-01T12:00:00-04:00" }
            ,
            "custom_fields": {
                "someCustomField": "value"
            },

        },
        "ingest": {
            "profile": "valid-ingest-profile-name"
        }
    },
    "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
    } 
  ]
}
Wenn Sie noch keine Anmeldeinformationen für Ihr Video Cloud-Konto eingerichtet haben, können Sie neue Anmeldeinformationen hinzufügen über die API.

Beachten Sie, dass Sie für die Videometadaten alle Felder einschließen können, die für die aufgelistet sind Video erstellen Betrieb der CMS-API.

Aufnahmeprofile definieren die Wiedergaben, die für den VoD erstellt werden. Zum valid-ingest-profile-name können Sie den Namen eines benutzerdefinierten oder Standard-Aufnahmeprofils für Ihr Konto ersetzen. Welche Standardprofile Sie haben, hängt davon ab, ob Ihr Konto für die dynamische Zustellung aktiviert ist oder nicht. Siehe die Standard-Aufnahmeprofile für die dynamische Bereitstellung und die kontextbezogene Codierung oder der Standard-Aufnahmeprofile für die Legacy-Transcodierung.

Notiere dass der profile Feld ist optional. Wenn Sie kein Profil angeben, wird das Standardprofil für das Konto verwendet.

An S3 senden

Fügen Sie eine Version des hervorgehobenen Ereignisses hinzu, damit eine VOD-Version des Ereignisses in Ihrem S3-Bucket gespeichert wird Ausgabe unten.

{
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 600,
  "ad_insertion": false,
  "outputs": [
    { OTHER_JOB_OUTPUT_OBJECT },
    { OTHER_JOB_OUTPUT_OBJECT_2 },
    {
      "url":"s3://YOUR_BUCKET/path/filename.mp4",
      "credentials": "YOUR_CREDENTIAL_LABEL",
      "notifications": [
        {
          "url": "http://httpbin.org/post?vodStateChange"
        },
        {
          "url": "http://httpbin.org/post?vodFinished",
          "event": "output_finished"
        }
      ]
    }
  ]
}
Ein Berechtigungslabel für einen gültigen Benutzernamen und ein gültiges Passwort für Ihren S3-Bucket sollte bei der Einrichtung Ihres Live-Kontos erstellt und an Sie gesendet worden sein. Wenn Sie es nicht haben, wenden Sie sich bitte an den Brightcove Support.

Erstellen Sie den VoD als offenen Videoclip

Eine alternative Möglichkeit, einen VOD für ein Live-Event zu erhalten, ist: Erstellen Sie einen VOD-Clip ohne Endzeit. Ausführliche Informationen finden Sie im Schnittdokument.

Einschränkungen

  • Wenn du stoppen (abbrechen) Der Live-Job, bevor der VoD erstellt wurde, wird nicht erstellt. Stoppen Sie stattdessen Ihren Encoder, um die Erstellung des VoD zu starten, und warten Sie, bis dieser abgeschlossen ist, bevor Sie den Job beenden.
  • Zum Statischer Einstiegspunkt strömt nur , Live to VOD kann nicht so eingestellt werden, dass der Stream nach Abschluss archiviert wird.

    Alternativen:

    • Benutzen Ausschnitt das gesamte Live-Event als VOD zu archivieren. Das Fenster zum Abschneiden ist für die Standarddauer nach einem Ereignis oder bis zur nächsten Aktivierung des SEP (je nachdem, welcher Zeitraum kürzer ist) verfügbar.
