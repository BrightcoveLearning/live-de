Live-API: Erstellen von VOD-Clips

In diesem Thema erfahren Sie, wie Sie Video-on-Demand-Clips (VOD) aus Ihren Live-Streams erstellen.

Überblick

Clips sind Videos, die aus einem Live-Stream extrahiert wurden. Sie können an einen S3-Bucket, eine FTP-Site oder ein Video Cloud-Konto gesendet werden. Der Clip wird als MP4-Video erstellt und in jedem Fall an das Ziel gesendet. Im Fall von Video Cloud wird der MP4 vom Aufgabesystem transcodiert, und welche Arten von Formatvarianten für das Video erstellt werden, hängt vom verwendeten Aufgabeprofil ab.

Definitionen für Clips werden mit dem erstellt /vods Endpunkt.

Clips können auf verschiedene Arten erstellt werden:

  • Mit stream_start_timecode und / oder stream_end_timecode definiert in SMPTE-Timecodes für das Live-Stream-Ereignis - Beachten Sie, dass dies erfordert, dass der Encoder Timecode-Informationen sendet
  • Mit start_time und/oder end_time definiert relativ zur Startzeit ( stream_start_time) des gesamten Livestream-Events
  • Mit start_time und / oder end_time definiert in Epoche (Unix) Zeit (in Sekunden)
  • Mit duration
  • Die VOD-API können verwendet werden mit verschlüsselt oder DRM-geschützt Arbeitsplätze. Derzeit unterstützt das Live-Modul dies nicht, wird es aber in einer zukünftigen Version tun.

Anmerkungen

  1. Um Clips so schnell wie möglich verfügbar zu machen, wird zuerst ein segmentgenauer Clip erstellt und dann durch einen rahmengenauen Clip ersetzt, sobald er verfügbar ist.
  2. Wenn Sie angeben duration Der resultierende Clip sieht wie folgt aus:
    • Wenn der Job aktiv ist und noch aktiv ist: (Anforderungszeit - Dauer) bis (Anforderungszeit)
    • Wenn der Job beendet ist: ( finished_at- Dauer) bis ( finished_at)
  3. Wenn Sie beide angeben start_time UND end_time::
    • Wenn der Job aktiv ist und noch aktiv ist: Solange das Zeitfenster der Epoche vollständig innerhalb des Zeitfensters liegt created_at und die Anforderungszeit, der Clip wird gemacht
    • Wenn der Job beendet ist: Solange das Zeitfenster der Epoche vollständig innerhalb des Zeitfensters liegt created_at und finished_at wird der Clip gemacht
  4. Clips von Live-Streams mit SSAI wird keine Anzeigen enthalten.
  5. Clips können bis zu 7 Tage nach einer Veranstaltung erstellt werden. Für SEP können sie bis zur nächsten Aktivierung oder 7 Tagen (je nachdem, was kürzer ist) erstellt werden.
  6. Die VOD-API fügt keine Inhalte außerhalb der im Stream vorhandenen Inhalte hinzu. Wenn Sie in einem 300 Sekunden langen Live-Stream 350 angeben, ist die Ausgabe 300 Sekunden lang.
  7. Sie müssen keinen DVR-fähigen Live-Stream verwenden, damit das Clipping funktioniert, da der Live-Stream während der Übertragung gespeichert wird und sofort und 7 Tage nach Ende des Ereignisses verfügbar ist.
  8. Brightcove Live-Clipping erzeugt nur einen Clip mit der gleichen Auflösung wie die Ausgabe mit der höchsten Auflösung. Es stimmt nicht mit der Auflösung der Quelleneingabe überein (es sei denn, dies entspricht der Ausgabe mit der höchsten Auflösung).

Clips können auch an mehrere Ziele gesendet werden:

  • EIN Video-Cloud Konto
  • Ein FTP-Server
  • Ein S3-Eimer

Wenn Sie einen Clip angeben, wird die Ausgabe muss enthalten entweder ein url Ziel oder ein videocloud Objekt, um die Erstellung des Videos und die Aufnahme des Clips detailliert zu beschreiben Video-Cloud.

Hinweis: Clips kann erstellt werden, während der Live-Stream ausgeführt wird. Dazu müssen Sie die Start- und Endzeiten des Clips in Epochenzeit oder relativ zu definieren Start Zeit des Live-Streams.

Referenzen

Wenn für das Ziel, an das Sie den Clip senden, Anmeldeinformationen erforderlich sind, können Sie diese mithilfe der Anmeldeinformationsvorgänge der Live-API erstellen. Sehen Anmeldeinformationen für die Live-API verwalten für mehr Details.

Beachten Sie, dass Sie beim Senden des Clips an ein mit dem Live-Konto verknüpftes Video Cloud-Konto nicht Fügen Sie die Anmeldeinformationen in den Clipping-Auftrag ein.

Endpunkt

Clips werden durch Senden von a erstellt POST Anfrage zu:

https://api.bcovlive.io/v1/vods

Anfragetext - Video-Cloud

Hinweis: Wir empfehlen dringend nicht Geben Sie das Aufnahmeprofil an, das zum Transkodieren des Clips verwendet werden soll, und verwenden Sie stattdessen das Kontostandardprofil. Dies verringert die Wahrscheinlichkeit von Fehlern beim Erstellen des Clips in Video Cloud

Beispiel 1: Start- / Endzeiten relativ zum Stream-Start

Der Anforderungshauptteil enthält Start- und Endzeiten sowie Details zum Senden des Clips. Hier ist ein Beispielanforderungstext, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud-Konto sendet:

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute und senden ihn an Video Cloud. Wir geben dem Clip einen Namen und ein paar Tags, ohne die Angabe zu machen Aufnahmeprofil zum Umcodieren, damit der Kontostandard verwendet wird, und Anweisung Video-Cloud zum Erfassen von Miniatur- und Posterbildern aus dem Clip während der Transcodierung.

Beispiel 2: Start- / Endzeiten in der Epochenzeit

Der Anforderungshauptteil enthält Start- und Endzeiten in der Epochenzeit sowie Details zum Senden des Clips. Hier ist ein Beispielanforderungstext, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud-Konto sendet:

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
    "outputs":[
      {
        "label": "60 secs - epoch time",
        "start_time": 1516652694,
        "end_time": 1516652754,
        "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
        "videocloud": {
          "video": {
          "name": "One Minute Clip",
          "tags": ["live", "clip"]
          },
            "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute zu einer bestimmten Epochenzeit (in diesem Fall 22. Jan 2018 um 08:24:54 GMT).

Beispiel 3: Dauer mit Startzeit relativ zum Stream-Start

Der Anforderungshauptteil enthält die Dauer und stream_start_time sowie Details zum Senden des Clips. Hier ist ein Beispielanforderungstext, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud-Konto sendet:

        },
        "ingest": {

        }
      }
    }
  ]
}

In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute, der 5 Minuten nach dem Start des Live-Streams beginnt.

Beispiel 4: Dauer ohne Start- oder Endzeit

Der Anforderungshauptteil enthält Start- und Endzeiten in der Epochenzeit sowie Details zum Senden des Clips. Hier ist ein Beispielanforderungstext, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud-Konto sendet:

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs - duration",
      "duration": 60,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
        "ingest": {
          "capture-images": true
        }
      },
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ]
}

In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute. Da wir keine Start- oder Endzeit angeben, wird der Clip aus den letzten 60 Sekunden des Live-Streams entnommen.

Beispiel 5: Verwenden stream_start_timecode und stream_end_timecode

Der Anforderungshauptteil enthält Start- und Endzeiten / Frames in HH: MM: SS: FF-Zeitcodes sowie Details zum Senden des Clips. Beachten Sie, dass der Encoder Timecodes senden muss, um Timecodes verwenden zu können. Hier ist ein Beispielanforderungstext, der einen Clip der 50-Minuten eines Streams erstellt und an ein Video Cloud-Konto sendet:

      "stream_start_timecode": "01:10:18:15",
      "stream_end_timecode": "01:11:08:15",
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",

          "name": "Fifty Minute Clip",

        },

        "capture-images": true
        }
      }
    },
    "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
  ]
}

Allgemeine Informationen zum Senden von Clips an Video-Cloud

Um zu sehen, welche Felder in die video und ingest Objekte, siehe die Dynamische Aufnahme-API Referenz.

Tun nicht umfassen die master Feld im ingest Objekt, da diese Informationen von der Live-API bereitgestellt werden.

Anfragetext - S3

Der Anforderungshauptteil enthält Start- und Endzeiten sowie Details zum Senden des Clips. Hier ist ein Beispielanforderungshauptteil, der einen Clip der dritten Minute eines Streams erstellt und an einen S3-Bucket sendet:

{
  "live_job_id":"{{job_id}}",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

In diesem Beispiel erstellen wir einen Clip mit einer Dauer von 30 Sekunden und senden ihn an einen S3-Bucket. Wir geben die Bucket-URL einschließlich des Dateinamens für den Clip und eine Zeichenfolge an, die den Namen der gespeicherten S3-Bucket-Anmeldeinformationen enthält. Die Anmeldeinformationen können vom Brightcove-Support für Ihr Konto eingerichtet werden.

API-Antwort

Die Antwort auf eine Anforderung zum Erstellen eines Clips enthält eine ID für den Job und die Bezeichnung, die Sie im Anforderungshauptteil festgelegt haben, sowie die Live-Job-ID:

{
  "vod_jobs": [
    {
      "jvod_id": "9582606c50d84be5ad4bc104f2aa3360",
      "label": "last 60 secs of live job"
    }
  ],
  "live_job_id": "88ba5d87b61a4ef3a6dddabd0c38d319"
}
