Live-DRM über die Live-API

In diesem Thema wird das Hinzufügen von DRM für Live-Streams über die LIve-API behandelt.

Einführung

DRM kann zu Live-Jobs hinzugefügt werden, indem das DRM Objekt im Anforderungshauptteil, wenn Sie den Job erstellen. Das DRM Objekt hat zwei Eigenschaften:

  • modes auf ein Array oder eines oder mehrere der folgenden Elemente setzen: "all", "fairplay", "playready", "widevine", "widevine: dash" oder "widevine: hls"
  • token_expires_in auf eine Zahl zwischen 43200 (Standard) und 31622400 (die Anzahl der Sekunden vor Ablauf des Tokens) eingestellt)

Rahmen mode to all führt dazu, dass alle verfügbaren DRM-Typen auf die Ausgänge angewendet werden. Standardmäßig wird DRM wie folgt auf Ausgänge angewendet:

  • FairPlay: Wird auf HLS-Ausgänge angewendet
  • Playready: Wird auf DASH-Ausgänge angewendet
  • Widevine: Wird auf HLS-Ausgänge angewendet

Widevine kann auf DASH angewendet werden Ausgänge anstelle von HLS. Um Widevine mit DASH zum Standard zu machen, ist eine Überschreibung auf Kontoebene erforderlich. Wenden Sie sich an den Support, wenn diese Überschreibung auf Ihr Konto angewendet werden soll. Im nächsten Abschnitt sehen Sie, wie Sie angeben können widevine:hls oder widevine:dash für bestimmte Jobs, unabhängig von der Standardeinstellung für Ihr Konto. Beachten Sie, dass je nach Konfiguration Ihres Kontos / Jobs die API-Antwort (wie die unten gezeigte) entweder Folgendes enthält widevine:hls oder widevine:dash Ausgänge, aber nicht beide.

Beachten Sie, dass in Ihrem Servicevertrag mit Microsoft Parallelitätsbeschränkungen für PlayReady definiert sind. Sehen dieses Microsoft-Dokument Informationen zum Verwalten der Parallelität. Wenn das Paralleleitungslimit für ein Video überschritten wird, wird es für Zuschauer über die maximale Anzahl hinaus nicht gespielt.

Erstellen Sie Live-Jobbeispiele

Alle DRM-Modi

Hier ist ein Beispiel für einen Anforderungshauptteil zum Packen von Ausgaben mit allen verfügbaren DRM-Typen:

    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 300,
    "ad_insertion": false,
    "static": false,

    "outputs": [{
        "label": "hls720p",

Wenn Sie für Ihr Konto die Standardeinstellung für die Verwendung von Widevine mit DASH festgelegt haben, können Sie Widevine-HLS-Ausgaben für einen bestimmten Job durch Angabe abrufen widevine:HLS in dem modes.

    "live_stream": true,
