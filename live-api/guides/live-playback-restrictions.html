Einführung

Wiedergabebeschränkungen können für einen Live-Job aktiviert werden, um den von DRM bereitgestellten Schutz zu erweitern, einschließlich:

  • Beschränkung der Wiedergabe auf einen bestimmten Benutzeragenten
  • Begrenzung der Anzahl der IP-Adressen
  • Begrenzung der Gesamtzahl der Benutzer

Um umfassende Einschränkungen zu erhalten, können Sie diese mithilfe der Wiedergaberechte-API konfigurieren. In diesem Fall würden Sie die zuordnen playback_rights_id mit einem Video. Einzelheiten finden Sie in der Einschränkungen auf Asset-Ebene Abschnitt der Übersicht: Brightcove-Wiedergabebeschränkungen Dokument.

Für Laufzeitbeschränkungen können Sie die JSON Web Token (JWT) -Ansprüche angeben. Diese Einschränkungen umfassen Stream-Parallelität und Gerätebeschränkungen. Einzelheiten finden Sie in der Laufzeitbeschränkungen Abschnitt der Übersicht: Brightcove-Wiedergabebeschränkungen Dokument.

Mit Live-Streams können Sie auch die DRM-CRT für einen Benutzer überschreiben. Einzelheiten finden Sie in der Schutz der Lizenzschlüssel Abschnitt der Übersicht: Brightcove-Wiedergabebeschränkungen Dokument.

Unabhängig davon, welchen Modus Sie auswählen, muss die Jobanforderung a enthalten videocloud.video Attribut, damit ein Video Cloud-Video erstellt und mit dem Live-Job verknüpft wird. Das playback_rights_id Das Attribut kann im Video Cloud-Video aktualisiert werden, um die Wiedergabebeschränkungen über das zu ändern CMS API.

Sie müssen auch Erstellen Sie ein JSON-Web-Token (JWT). mit den entsprechenden Ansprüchen. Das JWT muss in allen Lizenzserveranforderungen enthalten sein. Wenn Sie Ihren eigenen Player implementieren, müssen Sie diesen Wert entweder in a übergeben BCOV-Auth Header oder bcov_auth Abfrageparameter (zusätzlich zum DRM-Token).

Erstellen Sie einen Live-Job

Sie können Live-Jobs erstellen, die Wiedergabebeschränkungen verwenden. Einzelheiten zum Erstellen von Live-Streams finden Sie in der Erstellen Sie einen Live-Stream mit der Brightcove Live-API Dokument.

Neue Felder

Die folgenden neuen Felder werden verwendet, um einem Live-Job Wiedergabeeinschränkungen hinzuzufügen.

  • drm Objekt:

    require_playback_token(boolescher Wert)
    Playback Auth JWT ist erforderlich, wenn Sie eine Lizenz anfordern.
    check_playback_rights(boolescher Wert)
    Playback Auth JWT ist erforderlich, wenn Sie eine Lizenz anfordern. Verwendet die mit Video oder JWT verknüpfte Wiedergaberechten-ID, um Einschränkungen durchzusetzen.

  • videocloud.video Objekt:

    Beachten Sie, dass dieses Feld mithilfe der CMS-API zum Video hinzugefügt werden muss nach Sie erstellen den Live-Job, da er beim Erstellen des Videos nicht hinzugefügt werden kann.
    playback_rights_id(String)
    Verknüpft die angegebenen Wiedergabeeinschränkungen mit einem Video.

Fügen Sie dem Live-Job Einschränkungen hinzu

Nachdem Sie den Live-Stream erstellt haben, können Sie dem für den Live-Job erstellten Video mithilfe von Wiedergabeeinschränkungen hinzufügen CMS API.

CMS-API-Anfrage

Senden Sie eine, um dem Video die ID für die Wiedergaberechte hinzuzufügen Video aktualisieren Anforderung an die CMS-API mit dem folgenden Anforderungshauptteil:

{
	"playback_rights_id": "your playback_rights_id"
}

Liefern Sie den Stream
