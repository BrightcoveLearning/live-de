Einführung

Mit der Redundanzfunktion von Brightcove Live können Sie eine zuverlässige Leistung für Ihre Live-Ereignisse sicherstellen, indem Sie einen Sicherungsdatenstrom erstellen, auf den Live automatisch ein Failover durchführt, falls der erste Datenstrom nicht mehr funktioniert.

Beachten Sie, dass für alle Anforderungen an die Live-API die folgenden Header erforderlich sind:

API-Anforderungsheader
Schlüssel Wert Anmerkungen
X-API-KEY {your API Key} Ihr Schlüssel sollte beim Öffnen Ihres Brightcove Live-Kontos angegeben worden sein
Content-Type anwendung/json Technisch gesehen ist die content-type Der Header ist nur für Schreibanforderungen erforderlich, die einen Anforderungshauptteil enthalten, sollte jedoch Leseanforderungen keinen Schaden zufügen.

Erstellen Sie Ihre Live-Jobs

Für das redundante Setup müssen Sie 2 oder mehr Brightcove Live Jobs erstellen. Die einzige Voraussetzung für die Jobs ist, dass sie mit dem erstellt werden gleich output die Einstellungen. Der einfachste Weg, dies zu erreichen, besteht darin, einen Job mit den gewünschten Ausgabespezifikationen zu erstellen und dann die zu verwenden copy_outputs_from_job Parameter zum Erstellen der zusätzlichen Jobs.

Es gibt keine Einschränkung für die Regionen, in denen diese Arbeitsplätze geschaffen werden können. Es wird jedoch empfohlen, dass sie relativ nahe beieinander liegen.

Halten Sie nach dem Erstellen der Live-Jobs die Job-IDs für später fest. Der Anforderungshauptteil, mit dem Sie diese Jobs einer redundanten Gruppe hinzufügen (siehe die folgenden Abschnitte), sieht folgendermaßen aus:

    [
      {
        "job_id": "0b76bc73f92f46dc917bbe5061c0c633"
      },
      {
        "job_id": "0ae5a4a71dc54b3181af0f98ee407c27"
      }
    ]

Erstellen Sie die redundante Gruppe

Um eine redundante Gruppe zu erstellen, senden Sie eine POST Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups

Hier ist ein Beispiel für einen Anfragetext:

    {
      "ad_insertion": true,
      "processing_regions": ["us-west-2"],
      "storage_regions": ["us-west-2", "us-east-1"],
      "label": "Test RG",
      "live_dvr_sliding_window_duration": 1800
    }

Die folgende Tabelle enthält eine vollständige Liste der Felder für den Anforderungshauptteil. In einigen Fällen sind diese identisch mit Feldern, die zum Erstellen eines Live-Jobs verwendet werden. Siehe die Live API-Referenz Weitere Details zu den Feldern.

Feld Typ Erforderlich? Beschreibung
ad_insertion boolesch fakultativ Setzen Sie diesen Wert auf true, wenn dieser Stream SSAI-fähig sein soll
add_cdns Reihe fakultativ Array zusätzlicher CDN-Anbieter, die für die Manifestgenerierung verwendet werden sollen. Für jedes bereitgestellte CDN wird das Manifest entsprechend vorangestellt
drm Objekt fakultativ Noch nicht unterstützt

Sie können die Antwort mithilfe von filtern state param. Die zulässigen Werte sind:

  • cancelled
  • cancelling
  • deleting
  • disconnected
  • failed
  • finished
  • finishing
  • processing
  • standby
  • waiting

Da ist auch ein page_size Parameter, der auf eine Ganzzahl bis zu 1000 eingestellt werden kann. Der Standard page_size ist 10.

Fügen Sie der redundanten Gruppe Live-Jobs hinzu

Nachdem Sie eine redundante Gruppe erstellt haben, können Sie ihr Jobs hinzufügen, indem Sie a senden POST Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/jobs

Die Live-Job-IDs werden im Anforderungshauptteil wie folgt angegeben:

    [
      {
        "job_id": "0b76bc73f92f46dc917bbe5061c0c633"
      },
      {
        "job_id": "0ae5a4a71dc54b3181af0f98ee407c27"
      }
    ]

Es gibt einige zusätzliche optionale Eigenschaften für die Jobobjekte - die folgende Tabelle zeigt alle Felder:

Fügen Sie Anforderungsfelder für Live-Jobs hinzu
Feld Typ Erforderlich? Beschreibung
job_id Schnur erforderlich ID des Jobs, der zur Gruppe hinzugefügt werden soll. Wenn beides nicht playlist oder streams angegeben sind, alle outputs wird verwendet.
playlist Schnur fakultativ Bezeichnung der Wiedergabeliste, die als Ausgabe für den Stream verwendet werden soll. Wenn playlist ist definiert, streams muss Sein nicht definiert.
streams Reihe fakultativ Liste der Stream-Labels, die als Ausgaben für den Stream verwendet werden sollen. Wenn streams ist definiert, playlist muss Sein nicht definiert.

Die Erfolgsantwort für diese Anforderung gibt nur die redundante Gruppen-ID zurück.

Manuelles Hinzufügen von Cuepoints

Das manuelle Hinzufügen von Cuepoints zu einem Live-Stream mit Redundanz ist dem Hinzufügen von Cuepoints zu einem normalen Live-Stream sehr ähnlich. Um dies für eine redundante Gruppe zu tun, machen Sie a POST Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/cuepoint

Fügen Sie einen Anforderungshauptteil wie den folgenden hinzu:

    {
      "ad_server_data": {
        "subject": "wildlife"
      },
      "duration": 30,
      "timecode": "09:23:18:05"
    }

Wenn Sie das weglassen timecode Feld wird der Cuepoint sofort eingefügt.

Erzwingen Sie ein Job-Failover

Das Failover wird automatisch vom Brightcove Live-System verwaltet und sollte keine Intervention von Ihrer Seite erfordern. Wenn Sie aus irgendeinem Grund ein Failover auf einen anderen Job erzwingen möchten, ist es am einfachsten, den Encoder des zu stoppen on_air Job.

Sie können ein Failover auch mithilfe der API erzwingen, indem Sie a erstellen PUT Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/switch

Der Anfragetext sieht folgendermaßen aus:

    {
      "mode": "manual",
      "on_air": "0ae5a4a71dc54b3181af0f98ee407c27"
    }

Hier, mode wird entweder sein manual oder auto (die Standardeinstellung) und die on_air wird einen Wert von einem der haben sekundär Job-IDs.

Hinweis: Einmal in manual Modus, automatisches Job-Failover wird nicht auftreten. Sie Muss Stellen Sie den Modus wieder auf auto Damit das automatische Failover fortgesetzt wird. Sie können den On-Air-Job ändern, ohne auf manuell umzuschalten, indem Sie den Modus im Hauptteil weglassen und den Parameter für die Force-Abfrage hinzufügen ?force=true auf Ihre Anfrage. Dadurch wird der Dienst gezwungen, Jobs zu wechseln. Er kann jedoch jederzeit zurückwechseln, wenn Probleme erkannt werden.

Beenden eines redundanten Streams

Es gibt zwei Möglichkeiten, wie Sie Ihren redundanten Stream beenden können. Redundante Gruppen sind grundlos SEP-Streams. Sie können eine redundante Gruppe einfügen STANDBY Modus, indem Sie alle Jobs aus der redundanten Gruppe entfernen.

Verwenden Sie dazu a DELETE Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/jobs/{job_id}

für jeden Job, den Sie entfernen möchten. Beachten Sie dies, um den aktuellen Job zu entfernen on_air müssen Sie die hinzufügen ?force-true Abfrageparameter.

Die zweite Möglichkeit, den Stream zu beenden, besteht darin, die redundante Gruppe vollständig zu löschen, indem Sie a senden DELETE Anfrage zu:

    https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}

Einschränkungen

  • Live-Redundanz kann nicht kann mit DRM-geschützten Live-Streams verwendet werden.
