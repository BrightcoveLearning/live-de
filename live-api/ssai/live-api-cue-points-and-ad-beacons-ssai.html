Überblick

Serverseitige Anzeigenschaltung (SSAI) ermöglicht Ihnen die Anzeige von Anzeigen während eines Live-Streaming-Ereignisses zu bestimmten Zeiten. Allgemeine Informationen finden Sie in der Live-API: Serverseitige Anzeigenschaltung (SSAI) dokumentieren.

  • Bevor Sie SSAI verwenden können, muss es für Ihr Konto aktiviert sein. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu erhalten.

Cue-Punkte

Werbeunterbrechungen werden durch Cue-Points ausgelöst, die auf zwei Arten angegeben werden können:

  • Vom Encoder an Brightcove gesendet
  • Sofortige Cue-Points erstellt über die Live-API

Vom Encoder

Das Brightcove-Live-Liefersystem kann vom Encoder gesendete Cue-Points im AMF-Format interpretieren:

  AMFDataList
  [0]:onCuePoint
  [1]:{Obj[]:
    time: 1.9889, //Difference from PTS of THIS packet to the first PTS of the 1st video frame in the adbreak
    name: "scte35",
    type: "event",
    ad_server_data: "YWJjZGVmZ2g=",	// optional introduced by Brightcove. It is a base64 encoded json map of parameters e.g. {‘key’:’value’}
    parameters: {Obj[]:
      type: "avail_in",
      duration: 12.0
    }
  }

Anmerkungen:

  • Nur avail_in Typ-Cue-Punkte werden derzeit in der RTMP-Eingabe unterstützt.
  • SCTE-35-Cue-Points werden in RTP- und SRT-Eingängen unterstützt.
Siehe die Anhang unten für einen Screenshot eines Beispiel-Cue-Point-Setups für den Elemental-Encoder.

Manuelles Einfügen des Cue-Points

Sie können sofort Cue-Punkte erstellen, indem Sie die Live-API durch Senden einer POST Anfrage:

Methode POST
URL https://api.bcovlive.io/v1/jobs/Job_ID/cuepoint
Header X-API-KEY: your API KEY

Fügen Sie einen Anforderungshauptteil hinzu, der Folgendes angibt:

Feld Typ Beschreibung
duration Ganze Zahl Dauer der Pause in Sekunden.

Die Dauer des eingefügten Cue-Points muss sein mindestens doppelt so lang wie die Segmente in der Arbeit. Siehe die Dauer Beispiel.
timecode SMPTE-Format OPTIONAL: Ein Timecode im SMPTE-Format, HH: MM: SS: FF (FF = Frames), um anzugeben, wann eine Reihe von Variablen (Schlüssel / Wert-Paare) an den adServer übergeben werden soll.

Wenn er weggelassen wird, wird der Cue-Point sofort eingefügt.

Wenn Sie die timecode-Eigenschaft verwenden, muss der Encoder SMPTE-formatted (HH:MM:SS:FF) Timecode senden, der in der tc Eigenschaft gespeichert ist, über OnFI. Timecodes sind vom Beginn des Live-Streams an.
ad_server_data Objekt OPTIONAL: Die Schlüssel / Wert-Paare, die Sie übergeben, hängen vom verwendeten Ad-Server ab. Weitere Informationen finden Sie in der Dokumentation Ihres Ad-Servers und in der Targeting von Anzeigen mithilfe von Anzeigenmakros Sektion.

Beispiel für die Dauer

Die Dauer des eingefügten Cue-Points muss sein mindestens doppelt so lang wie die Segmente in der Arbeit.

Zum Beispiel Einfügen von a 10 Sekunden Cue-Point in einem Job mit "segment_seconds"=4 wird gut funktionieren. Einfügen des gleichen Cue-Points in einen Job mit "segment_seconds"=6 führt zu folgendem Fehler:

  "error": "The parameter duration should be greater than
    or equal to (2 * target duration) of the job"
 

Beispielanforderungshauptteil

  {
    "duration": 30,
    "timecode": "15:50:49:16",
    "ad_server_data" : {
    "adbreakid": 12312
    "breaktheme": "fitness"
    }
  }

Anmerkungen

  1. Software-Encoder wie Wirecast und OBS unterstützen den Sende-Timecode über nicht OnFI Pakete im RTMP-Stream
  2. Elementare Hardware-Encoder unterstützen den Sende-Timecode über OnFI Pakete im RTMP-Stream

Beispielantwort

  {
    "id": "Job_ID",
    "cue_point": {
      "id": "adBreak-2f58393ada1442d98eca0817fa565ba4",
      "duration": 30,
      "accuracy": "segment", [Can be segment or frame  ]
      "inserted_at": "2017-07-21T09:30:46.307Z" [ Time when the cue point was inserted in the stream]
    },
  }

Leuchtfeuer

Beacons sind Datenpunkte bei der Wiedergabe, die an Analysen von Drittanbietern gesendet werden, um zu verfolgen, ob und wie viele Anzeigen abgespielt wurden. In diesem Abschnitt werden wir uns die Beacon-Typen ansehen, die mit der Live-API festgelegt werden können, und Variablen, die zur Bereitstellung der Daten verwendet werden können. Im nächsten Abschnitt werden die API-Anforderungen beschrieben, die zum Erstellen und Verwalten von Beacon-Sets verwendet werden.

Leuchtfeuertypen

Leuchtfeuertypen
Leuchtfeuertyp Beschreibung
Load Wird einmal pro Sitzung ausgelöst und nur ausgelöst, wenn ein Manifest der obersten Ebene angefordert wird
Play Der Inhalt wurde angefordert und das erste Segment zurückgegeben
Heartbeat Zieldauer (Segmentsekunden)
AdStart Einzelanzeige gestartet
AdFirstQuartile Erstes Anzeigenquartil (25%)
AdMidpoint Zweites Anzeigenquartil (50%)
AdThirdQuartile Drittes Anzeigenquartil (75%)
AdComplete Einzelanzeige abgeschlossen
AdBreakStart Die Werbeunterbrechung hat begonnen
AdBreakComplete Werbeunterbrechung ist beendet

SCTE-35-Anzeigenvariablen

Das Gesellschaft der Kabeltelekommunikationsingenieure (SCTE) Definiert einen Standard für das dynamische Einfügen von Anzeigen für Live-Streams. In der folgenden Tabelle sind die SCTE-35-Anzeigenkonfigurationsvariablen zusammengefasst.

Die SCTE-35-Anzeigenparameter werden derzeit sowohl mit RTP (Real-Time Transport Protocol) mit oder ohne FEC (Forward Error Correction) als auch mit SRT (Secure Reliable Transport) -Eingangsquellen unterstützt, die auf Transport Stream (TS) basieren. Beachten Sie, dass FEC für RTP dringend empfohlen wird.

Diese Anzeigenparameter sind bei Eingabestreams, die das Real-Time Messaging Protocol (RTMP) verwenden, nicht verfügbar.
SCTE-35-Anzeigenkonfigurationsvariablen
Variable
Beschreibung
scte35_eventID
eine eindeutige Ereignis-ID, die mit einer SCTE35-Nachricht übergeben wurde.
scte35_programID
Eine eindeutige Programm-ID, die mit einer SCTE35-Nachricht übergeben wurde.
scte35_availNum
Eine ID für eine bestimmte Spleißzeit, die für Anzeigen verfügbar ist und über eine SCTE35-Nachricht gesendet wird.
scte35_breakDuration
Die Unterbrechungsdauer für die Werbeunterbrechung in Form von Ticks des 90-kHz-Takts des Programms wurde mit einer SCTE35-Nachricht übergeben.
scte35_spliceTime
Die Spleißzeit für eine Werbeunterbrechung, ausgedrückt als Ticks des 90-kHz-Takts des Programms, wurde mit einer SCTE35-Nachricht verstrichen.

Beacon-Sets verwalten

Dieser Abschnitt enthält Details zu den API-Anforderungen zum Verwalten von Beacon-Sets. Im vorherigen Abschnitt finden Sie Informationen zu Beacon-Typen und -Variablen.

Um einem Live-Job ein Beacon-Set hinzuzufügen, erstellen Sie zuerst das Beacon-Set und geben Sie dann die ID ein, wenn Sie den Job wie folgt erstellen:

  {
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 30,
  "ad_insertion": true,
  "beacon_set": "beacon_set_id", ...

Erstellen Sie ein Beacon-Set

Senden Sie a, um ein Beacon-Set zu erstellen POST Anfrage:

Methode POST
URL https://api.bcovlive.io/v1/ssai/beaconsets
Header X-API-KEY: your API KEY

Beispielanforderungshauptteil

  {
    "account_id": "User's Account ID [Optional]",
    "beacon_urls": [
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/load?position=load&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
        "beacon_type": "Load"
      },
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/play?position=play&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
        "beacon_type": "Play"
      }
    ]
  }
Die account_id Feld ist die Live-Konto-ID. Wenn es weggelassen wird, wird die Konto-ID des anfragenden Benutzers verwendet.

Beispielantwort

  {
    "beacon_set": {
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/load?position=load&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
      "beacon_type": "Load"
    },
    {
      "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/play?position=play&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
      "beacon_type": "Play"
    }],
    "beacon_set_id": "Inserted Beacon Set ID",
    "account_id": "USER's ACCOUNT ID"
    }
    "inserted": true
  }

Aktualisieren Sie ein Beacon-Set

Das Aktualisieren eines Beacon-Sets ähnelt dem Erstellen eines Beacon-Sets. Sende ein PUT Anfrage:

Methode PUT
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
Header X-API-KEY: your API KEY

Beispielanforderungshauptteil

  {
    "account_id": "User's Account ID [Optional]",
    "beacon_urls": [
      {
      "beacon_url": "https://myserver.com/beaconRX/load",
      "beacon_type": "Load"
      },
      {
      "beacon_url": "https://myserver.com/beaconRX/play",
      "beacon_type": "Play"
      }
    ]
  }
Die account_id Feld ist die Live-Konto-ID. Wenn es weggelassen wird, wird die Konto-ID des anfragenden Benutzers verwendet.

Beispielantwort

  {
    "beacon_set": {
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID",
      "beacon_urls": [{
        "beacon_url": "https://myserver.com/beaconRX/load",
        "beacon_type": "Load"
        },
        {
        "beacon_url": "https://myserver.com/beaconRX/play",

Löschen Sie ein Beacon-Set

Um ein Beacon-Set zu löschen, senden Sie abschließend a DELETE Anfrage:

Methode DELETE
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
Header X-API-KEY: your API KEY

Beispielantwort

Die Antwort sieht folgendermaßen aus:

  {
    "beacon_set_id": "Beacon set ID",
    "deleted": true
  }
