Überblick

SSAI ermöglicht Ihnen die Anzeige von Anzeigen während eines Live-Streaming-Ereignisses zu bestimmten Zeiten. Die Hauptmerkmale umfassen:

  • Fügen Sie Anzeigen mit Cue-Points ein, die von Ihrem Encoder gesendet werden, oder erstellen Sie einen Instant-Cue-Point mit dem Live-API
  • Nehmen Sie "Slate" -Anlagen auf, um nicht genutzte Werbezeiten zu füllen
  • Vermeiden Sie Werbeblocker mit Anzeigen, die auf der Serverseite in den Live-Stream eingebunden sind
  • Bevor Sie SSAI verwenden können, muss es für Ihr Konto aktiviert sein. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu erhalten.

Gehen Sie folgendermaßen vor, um einen Live-Stream mit serverseitigen Anzeigen zu erstellen:

  1. Überprüfen Sie die allgemeinen Informationen zur Live-API
  2. Erstellen Sie eine Live-Anzeigenkonfiguration. Sie können dies auch in tun Video Cloud Studio. In den folgenden Abschnitten finden Sie Details zum Verwalten Ihrer Anzeigenkonfigurationen mithilfe der Live-API.
  3. Erstellen Sie Schiefer-Assets. Wenn Sie eine Benutzeroberfläche bevorzugen, können Sie Slates in Studio verwalten.
  4. Optional: Fügen Sie Cue-Points und Ad Beacons ein.
  5. Jetzt sind Sie bereit dazu Erstellen Sie einen Live-Stream mit der Live-API. Wenn Sie Studio bevorzugen, können Sie dies tun Implementieren Sie serverseitige Anzeigen im Live-Modul

Weitere Informationen zu benutzerdefinierten Headern, Anzeigenkonfigurationsvariablen, DFP- und Anzeigenmakros finden Sie im Rest dieses Dokuments.

Allgemeine Information

Die folgenden Informationen beziehen sich auf alle Live-APIs mit SSAI-Anforderungen.

Basis-URL

Die Basis-URL für die Live-API mit SSAI lautet:

    https://api.bcovlive.io/v1/ssai

Authentifizierung

Die Authentifizierung für Anforderungen erfordert einen Header mit einem API-Schlüssel:

    X-API-KEY: your API KEY

Wenden Sie sich an Ihren Account Manager, um einen API-Schlüssel für Ihr Konto zu erhalten.

Schiefer verwalten

Slates sind Ihre eigenen Assets, mit denen Sie nicht genutzte Werbezeiten füllen. Sie können Slates verwenden, um eine "Be Right Back" -Nachricht oder einen beliebigen Inhalt bereitzustellen.

Nachfolgend finden Sie Details zu den API-Anforderungen zum Hinzufügen und Verwalten von Slate-Assets.

Schieferobjekt hinzufügen

Senden Sie a, um ein neues Slate-Medienquellen-Asset aufzunehmen POST Anfrage:

Methode POST
URL https://api.bcovlive.io/v1/ssai/slates
Header X-API-KEY: your API KEY

Hinweise zu DFP

Wenn Sie Anzeigen von DFP erhalten, sollten Sie Folgendes beachten, um Probleme zu vermeiden.

Anzeigen-Tag

Stellen Sie beim Erstellen eines Anzeigen-Tags für Live sicher, dass Sie die entsprechenden Richtlinien befolgen und einschließen /live . Siehe das DFP-Dokument Erstellen Sie manuell ein Master-Video-Tag für alle Details.

Parallelität

Wenn Sie eine hohe Parallelität erwarten, empfehlen wir Ihnen, mit Ihrem DFP-Kontoteam zu sprechen.

Einzel- / Mehrfachanzeigenantworten

Die singleadresponse und multiadresponse Parameter werden derzeit nicht verwendet von SSAI.

Live SSAI führt nur einen einzigen Ad-Server-Aufruf aus und erwartet, dass die Antwort alle Anzeigen für die Unterbrechung enthält, mit der Ausnahme, dass sie allen erforderlichen Anzeigen-Wrappern mit einem Limit von 5 Weiterleitungen pro Anzeige folgt. Die folgenden Anzeigenantwortformate werden akzeptiert und wie folgt analysiert:

  • VAST - Einzelne Antwort oder eine Ansammlung von Anzeigen in einer einzelnen Antwort
  • DFP-Anzeigenregeln - Aggregiert alle verfügbaren Anzeigen in der Antwort, einschließlich definierter Anzeigen vor, während und nach dem Roll
  • Smart XML - Aggregiert alle verfügbaren Anzeigen in der Antwort, einschließlich definierter Anzeigen vor, während und nach dem Roll

Benutzerdefinierte Header für Anzeigenanfragen

Die SSAI Plattform kann benutzerdefinierte Header mit den Anzeigenaufrufen und allen von der Back-End-Plattform verwendeten Beacons übergeben. Einige Ad-Server wie VideoPlaza erfordern benutzerdefinierte Header.

Benutzerdefinierte Header werden als Satz von Schlüssel-Wert-Paaren in einem angegeben ad_configuration_headers Objekt, das Teil des ist application_ad_configuration (siehe die Erstellen Sie eine Anzeigenkonfiguration Sektion).

Anmerkungen

  • Standardheader werden standardmäßig behandelt, z. B.:
    • X-Forwarded-For
    • X-Device-User-Agent
  • Header-Werte können die verwenden Anzeigenkonfigurationsvariablen
  • Header-Werte können auch statische Zeichenfolgen sein
Header werden an alle nachgelagerten Anfragen weitergegeben, einschließlich Werbeverpackungen oder Impressionen/Quartil, daher sollten Sie sie mit Vorsicht verwenden.

Targeting von Anzeigen mithilfe von Anzeigenmakros

Wenn du Erstellen Sie eine Anzeigenkonfiguration Normalerweise sieht Ihr Anzeigen-Tag folgendermaßen aus:

      https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}}&
      num={{random.int32}}&ses={{session.session_id}}&IDFA={{deviceid}}&
      sitesection={{sitesection}}&breakid={{breakid}}&breaktheme={{breaktheme}}

Die Elemente in den doppelten geschweiften Klammern sind Variablen, auch Anzeigenmakros genannt, die Brightcove Live beim Senden der Anzeigenanforderung durch Werte ersetzt, sofern vorhanden.

Anmerkungen:

  • Welche Variablen nützlich sind, hängt vom Ad-Server ab.
  • Der Brightcove-URI codiert jeden Wert, der in die Ad-Server-URL eingefügt wird.
  • Wenn für einen Schlüssel mehrere Werte vorhanden sind, können die codierten Variablen vom Ad-Server nicht gelesen werden. Dies liegt an der URI-Codierung.

Anzeigenmakrowerte können auf folgende Arten bereitgestellt werden:

  • Header-Informationen verwenden
  • URL anhängen
  • URLs dynamisch hinzufügen
  • Verwenden manueller Cue-Points

Header-Informationen verwenden

Header-Informationen, detailliert in der Anzeigenkonfigurationsvariablen Abschnitt oben, ist für jede Anfrage verfügbar. Geben Sie einfach die gewünschten Variablen mit den entsprechenden Schlüsselnamen in Ihrer Anzeigenkonfiguration an.

URL anhängen

Zusätzliche Sitzungswerte können wie folgt an die URL für den Live-Stream angehängt werden:

      http://playback.bcovlive.io/e058d9f2737e18/us-west-2/NA/
      playlist.m3u8?deviceid=123456&sitesection=services"

URLs dynamisch hinzufügen

Sie können URLs mithilfe von Javascript und der Brightcove Player-API dynamisch hinzufügen:

      <!DOCTYPE html>
      <body>
        <video
        id="myPlayerID"
        data-video-id="5975635167001"
        data-account="3737230870001"
        data-player="tIG7lVKBm"
        data-embed="default"
        data-application-id
        class="video-js"
        controls
        width="640"
        height="360"></video>
        <script src="//players.brightcove.net/3737230870001/tIG7lVKBm_default/index.min.js"></script>

        <script type="text/javascript">
        var player = videojs("myPlayerID");

        player.one("loadstart", function(){
          var sourceUrl = player.currentSource();
          console.log(sourceUrl);
          var liveUrlWithParams = sourceUrl.src + "?player_width={width}&player_height={height}&deviceid={deviceid}";

          player.src([{
            "type": "application/vnd.apple.mpegurl",
            "src": liveUrlWithParams
          }]);
        });
      </script>

      </body>

Beachten Sie, dass die Schlüsselnamen in diesem Beispiel den Variablennamen im oben gezeigten Anzeigen-Tag entsprechen.

Anzeigenkonfigurationsvariablen

Anzeigenkonfigurationsvariablen, auch als Beacons bezeichnet, können in Anforderungen zum Verwalten von Anzeigenkonfigurationen verwendet werden. Einzelheiten finden Sie in der Live-API: Cue Points und Ad Beacons mit SSAI Dokument.

Verwenden manueller Cue-Points

Werte für bestimmte Werbeunterbrechungen können mit der manuellen Anforderung zum Einfügen von Cue-Points gesendet werden. Einzelheiten finden Sie in der Live-API: Cue Points und Ad Beacons Dokument.

Bekannte Probleme

  • SSAI erfordert, dass das Live-Streaming-Video über eine Audiospur verfügt.
  • Wenn der zurückgegebene VAST dieselbe Anzeigen-ID hat, fordert der Server den Anzeigeninhalt nicht an, obwohl die Anzeigen-URL dynamische Variablen verwendet, um eine eindeutige URL zu erstellen. Das macht nicht Bewerben Sie sich, wenn Sie DFP für Anzeigen verwenden.
  • Mit DFP können Sie zwar dieselbe Anzeigen-ID verwenden, es muss jedoch eine andere Creative-ID vorhanden sein. Mit anderen Worten, Sie können die MediaFile nicht einfach austauschen.

  • Wenn eine Anzeigenunterbrechung eine Dauer hat, die kürzer als die MAX-Dauer der Anzeigen-URL ist (min_ad_duration = 0 & max_ad_duration = 30000), wird die Anzeige nicht abgespielt, wenn die Anzeige länger als die Anzeigenunterbrechung zurückgegeben wird.

  • VPAID oder anklickbare Anzeigen werden nicht unterstützt für Brightcove Live-SSAI.

  • Wenn für eine Werbeunterbrechung die verbleibende Zeit kürzer als die Segmentsekunden des Streams ist und eine Tabelle angezeigt wird, wird die Liste für ihre Segmentdauer angezeigt und die tatsächliche Werbeunterbrechung ist länger als erwartet.

  • Dies kann dazu führen, dass eine der folgenden Anzeigenunterbrechungen um die Latenzzeit verkürzt wird, um zu versuchen, die Sitzung wieder auf den Live-Rand zu bringen.

  • Wenn die Anzeige zum ersten Mal von unserem System gesehen wird, wird sie erst abgespielt, wenn sie transkodiert und zur Auslieferung bereit ist.
  • VMAP for Live SSAI wird derzeit nicht unterstützt.
