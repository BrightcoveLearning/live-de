Live-DRM

In diesem Thema erfahren Sie, wie Sie den Inhaltsschutz für Ihren Live-Stream aktivieren.

Aktivieren von DRM für Live-Inhalte

Hinweis:

Standardmäßig wird DRM wie folgt auf Ausgänge angewendet:

  • Ihr Live-Konto muss zuerst aktiviert werden von Brightcove Support, um DRM zu verwenden.

  • Verwenden Sie eine der folgenden Optionen, um ein DRM-fähiges Ereignis zu veröffentlichen:

    • Ein DRM-fähiger Brightcove Player v6.37.1 oder höher
    • Das Brightcove Native SDK für iOS v6.5.0 oder höher

    • Das Brightcove Native SDK für Android v6.11.0 oder höher

  • Wenn Sie die DRM-Standardeinstellungen überschreiben möchten, können Sie dies über die API tun oder sich an den Brightcove-Support wenden, um sie in der Live-Plattform zu ändern.
    • FairPlay: Wird auf HLS-Ausgänge angewendet
    • Playready: Wird auf DASH-Ausgänge angewendet
    • Widevine: Wird auf HLS-Ausgänge angewendet[1-1]

Anmerkungen

  • [1-1] Widevine kann anstelle von HLS auf DASH-Ausgaben angewendet werden. Um Widevine mit DASH zum Standard zu machen, ist eine Überschreibung auf Kontoebene erforderlich. Wenden Sie sich an den Support, wenn diese Überschreibung auf Ihr Konto angewendet werden soll. Verwendung der Live-API du kannst auch angeben widevine:hls oder widevine:dash für bestimmte Jobs, unabhängig von der Standardeinstellung für Ihr Konto. Beachten Sie, dass je nach Konfiguration Ihres Kontos / Jobs die API-Antwort (wie die unten gezeigte) entweder Folgendes enthält widevine:hls oder widevine:dash Ausgänge, aber nicht beide.

Gehen Sie folgendermaßen vor, um DRM für Ihre Live-Inhalte zu aktivieren:

  1. In ERWEITERTE OPTIONEN> Inhaltsverschlüsselung. Wählen Digital Rights Management (DRM) aus der Dropdown-Liste Verschlüsselung.
  2. Ein zusätzliches Token-Ablauf Die Option wird angezeigt, mit der Sie die Zeit in Sekunden mit einem minimalen Ablaufzeitraum von 900 Sekunden und einem Maximum von 2592000 Sekunden (der letzte entspricht 30 Tagen) auswählen können.
  3. Im Ereignisinformationsbereich können Sie die DRM-URLs für Fairplay und Widevine anzeigen.
  4. Um ein DRM-fähiges Ereignis zu veröffentlichen, klicken Sie in der linken Navigation auf Veröffentlichen und Einbetten> Player-Konfiguration.
    • Im Live Player-Optionen Wählen Sie einen DRM-fähigen Player aus. (Für die Live-DRM-Wiedergabe müssen Sie eine auswählen oder erstellen DRM-fähiger Player v6.37.1 oder höher).
    • Wenn Sie keinen DRM-Player zur Verfügung haben, gehen Sie zu Player-Modul > Player auswählen oder erstellen> Wiedergabe in der linken Navigation. Wähle aus "DRM aktivieren" Möglichkeit. (Die Wiedergabe von Player für Live-DRM muss Version 6.37.1 oder höher sein).
  5. Zurück im Live-Modul können Sie Ihren DRM-fähigen Player auswählen und Ihr DRM-Ereignis veröffentlichen.
