Aktivieren der Live-Redundanz

Anmerkungen:

  • Jedes Konto hat ein SEP-Limit (Static Entry Point) von 3, mit Ausnahme von us-west-2, das ein Limit von 10 hat. Wenn Sie eine Erhöhung benötigen, wenden Sie sich bitte an den Brightcove-Support.
  • Wenn Sie Redundanz mit SSAI verwenden, müssen Sie Anzeigen gleichzeitig in alle Jobs in der Gruppe einfügen.

Die Live-Redundanzfunktion wird aktiviert, indem Sie Ihrem Ereignis in der Brightcove Live-Plattform zwei oder mehr Streams hinzufügen.

Erstellen Sie zuerst ein Live-Ereignis. (Weitere Informationen zum Erstellen eines Live-Events finden Sie hier Klicke hier).

  1. Gehe zum Stream-Optionen Mit diesen Optionen können Sie die Region Ihrer Streams benennen und auswählen. Klicken Sie in die (+) Fügen Sie einen Backup-Stream hinzu Taste.

    Wenn die Schaltfläche Backup-Stream hinzufügen nicht angezeigt wird, wurde das Backup-Streaming für Ihr Konto nicht aktiviert.

    Hinweis: Bitte wenden Sie sich an den Brightcove Support, um diese Option in Ihrem Konto zu aktivieren.
  2. Durch Klicken auf die Fügen Sie einen Backup-Stream hinzu Klicken Sie auf die Schaltfläche, um einen Sicherungsdatenstrom (Redundanzstrom) für Ihr Live-Video hinzuzufügen. Sie können so viele addieren, wie Ihr Kontolimit unterstützt.
    Möglicherweise haben Sie redundante Gruppen, die auf Ihr Kontolimit angerechnet werden. Sobald ein Ereignis mit redundanten Streams abgeschlossen oder abgebrochen wurde, ist es im Live-Modul nicht mehr sichtbar. Solche Ereignisse können jedoch über die Live-API aufgelistet und gelöscht werden. Sehen Live-API: Redundante Gruppen.
  3. Benennen Sie Ihre primären und Backup-Streams und wählen Sie eine Region aus, die dem Ort Ihres Encoders am nächsten liegt. Standardmäßig ist die Primärstrom Die Region ist auf Oregon (US-West) eingestellt, was zu schlechten Ergebnissen führt, wenn Sie von einem anderen Kontinent streamen.

    Das Referenz ID ist eine optionale Kennung für den internen Gebrauch und muss eindeutig sein (maximal 150 Zeichen).

    Richten Sie Ihr Live-Event wie gewohnt ein. Weitere Informationen zum Erstellen eines Live-Events finden Sie hier Klicke hier.

  4. Alternativ können Sie auch einen neuen Stream aus dem Kontrollraumfenster hinzufügen:
  5. Fügen Sie Ihre Backup Stream-Informationen und Ihre Region hinzu.
  6. Im Kontrollraum können Sie die sehen Endpunkt und Stream Name Details zu jedem Ihrer Streams.
  7. Sie können alle Stream-Informationen in der überprüfen Kontrollraum.

Hinweis: Wenn Sie ein Einzelstromereignis erstellt und im Kontrollraum einen Redundanzstrom hinzugefügt haben, müssen Sie den eingebetteten Code Ihres Players erneut veröffentlichen, um von der Redundanz zu profitieren.

Ausschnitt

  1. Video-Clipping ist für jeden Ihrer Streams verfügbar, indem Sie einfach den gewünschten Stream aus dem Dropdown-Menü auswählen. Informationen zum Erstellen eines Clips finden Sie unter Erstellen eines Videoclips mit dem Live-Modul.
  2. Zweite Stream-Auswahl.

Überwachung

Klicken Sie auf die Registerkarte Überwachung unter dem Vorschau-Player, um Überwachungsinformationen anzuzeigen. Die Überwachung ist für jeden Ihrer Streams verfügbar, indem Sie einfach den gewünschten Stream aus dem Dropdown-Menü auswählen.

Die Überwachungsinformationen sind während eines Ereignisses und bis zu 1 Stunde nach Ende des Ereignisses verfügbar.

Löschen einer redundanten Gruppe

Mit einer erstellten redundanten Gruppe:

  1. Klicken Stream abbrechen von einem Arbeitsplatz der Gruppe.
  2. Klicken Stoppe diesen Stream aus einem zweiten Job der Gruppe.
  3. Gehe zum Ereignisse verwalten Seite und wählen Sie die Vollendet Tab.
  4. Das Redundanz-Ereignis sollte in enden Leerlauf Zustand.
  5. Klicken Nah dran um das Redundanzereignis zu entfernen.

Einschränkungen

  • Live-Redundanz kann nicht kann mit DRM-geschützten Live-Streams verwendet werden.
